# CWT 命題工作平臺 (官方版)

## 網站介紹

「CWT 命題工作平臺」官方版是一個基於 Blazor Server 技術開發的進階線上命題與審核管理系統。此平台旨在提供一個高效、安全且功能全面的環境，支援全民中文檢定（CWT）的試題從命製、多階段審核、修訂到最終入庫的完整生命週期管理。系統針對不同使用者角色（如系統管理員、命題教師、審核委員等）提供精細的權限控制，並整合了數據分析、相似度比對等多項功能，以確保試題品質、提升工作效率並優化使用者體驗。

## 主要功能

*   **高安全性使用者認證與權限管理**：基於 ASP.NET Core Cookie Authentication，提供安全的登入機制，並實現精細的角色與模組權限控制。
*   **多階段試題審核流程**：支援「命題」、「互審」、「專審」、「總審」等多個審核階段，並協調各階段流轉，確保試題品質與嚴謹性。
*   **即時試題狀態追蹤與通知**：透過 SignalR 實現即時通訊，確保各角色能即時獲取試題狀態更新、公告與急件提醒，提升協作效率。
*   **靈活的試題內容管理**：支援多種題型（如單選題、閱讀題組、聽力測驗等），提供豐富的編輯器（Quill Editor）與預覽功能，確保試題內容的完整性與呈現效果。
*   **數據概覽與報表匯出**：提供關鍵數據概覽，並支援將試題報表匯出為 Excel 格式（透過 NPOI 函式庫），方便管理者掌握整體命題進度與成果。
*   **相似度分析（4-gram 試題比對）**：以 4-gram + Jaccard 演算法即時比對跨梯次題庫，依題型加權計算相似度分數並分級示警，協助命題教師與審核委員判斷試題內容的原創性（詳見下方說明）。
*   **審後修訂機制**：支援管理員在三審結束後對試題進行再編輯與決策調整。
*   **聘書自動繪製與下載**：支援自動生成聘書並打包下載，簡化行政流程。

### 相似度分析（4-gram 試題比對）細節

為了防止同梯次重複出題、抄錄歷史題或教師自抄，系統內建一套**純 C# 實作、零外部 AI 依賴**的中文試題相似度比對引擎。當命題教師或審核委員在編輯／審題畫面點擊「比對相似題」時，後端即時計算並回傳最相似的前幾道題目，過程不寫入資料庫，純記憶體運算、回應快速。

**演算法核心 — 4-gram + Jaccard 係數**

1.  **文字正規化**：先移除富文本的 HTML 標籤、解碼 HTML 跳脫字元，再將全形英數標點轉為半形、移除所有空白，避免「下列 何者」與「下列何者」被誤判為不同內容。
2.  **切分 4-gram**：將正規化後的文字以 4 個字為一個滑動窗切成片語集合。例如「下列何者為臺灣」會切成 `下列何者`、`列何者為`、`何者為臺`、`者為臺灣`。選用 4-gram 是因為對中文檢定題目的辨識度與容錯度最為平衡。
3.  **計算 Jaccard 相似度**：取兩題片語集合的「交集大小 ÷ 聯集大小」，換算為 0～100 分。兩題用字重疊度越高、分數越高。

**依題型加權比對**

不同題型的資訊重心不同，系統針對 7 種題型套用各自的欄位權重，再加權平均出總分：

| 題型 | 加權方式 |
| :--- | :--- |
| 一般 / 精選題 | 題幹 70% + 四選項 30% |
| 短文題組 | 文章 60% + 子題題幹 40% |
| 長文題目 | 文章 90% + 標題 10% |
| 閱讀題組 | 文章 50% + 子題題幹 30% + 子題選項 20% |
| 聽力測驗 | 語音內容 70% + 題目 20% + 四選項 10% |
| 聽力題組 | 語音逐字稿 70% + 子題題幹 30% |

**分級示警門檻**

系統依分數分為三級，搭配顏色標示讓使用者快速判斷風險：

| 分數 | 判定 | 意義 |
| :--- | :--- | :--- |
| ≥ 80 | 確認重複 | 高度雷同，需特別留意 |
| ≥ 60 | 相似度高 | 用字大量重疊，建議檢視 |
| ≥ 40 | 安全 | 部分相似，僅供參考 |

**比對範圍**：比對時自動鎖定**相同題型、相同等級**的題目，並**跨越所有梯次**（讓教師看到跨年度可能重複的舊題），同時排除題目本身、命題草稿、已不採用、結案未採用及已刪除的題目，確保比對結果聚焦於有效題庫。

## 技術棧

*   **後端框架**：ASP.NET Core 10 (Blazor Server，InteractiveServer 渲染模式)
*   **程式語言**：C#
*   **資料庫存取**：Dapper (輕量級 ORM)、Microsoft.Data.SqlClient
*   **身分認證**：ASP.NET Core Cookie Authentication（搭配 Dapper 自訂 MT_Users 帳號表，未採用 EF Core 綁定的 Identity）、PBKDF2 密碼雜湊與帳號鎖定機制
*   **即時通訊**：ASP.NET Core SignalR
*   **前端框架**：Blazor Components、HTML5、CSS3 (Tailwind CSS v4)、JavaScript (ES6+)
*   **前端函式庫**（皆透過 JS Interop 整合）：
    *   **Quill**：富文本編輯器，用於題幹、選項、解析等富文本內容輸入。
    *   **SweetAlert2**：對話框與提示訊息（確認框、Toast 通知）。
    *   **ApexCharts**：命題儀錶板的數據圖表視覺化。
    *   **Font Awesome 6 / Noto Sans TC**：離線圖示與中文字型。
*   **後端函式庫**：HtmlSanitizer (HTML 內容消毒，防範 Stored XSS)、NPOI (Excel 檔案讀寫)
*   **效能快取**：IMemoryCache (角色權限、題型目錄等短期快取)
*   **開發工具**：Visual Studio 2026
