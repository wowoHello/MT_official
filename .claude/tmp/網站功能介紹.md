> 網站功能介紹篇
> 網頁色調：**莫蘭迪配色**
> 若有必要額外增加 **說明語言**

## 登入頁
- 輸入帳號
- 輸入密碼
- **忘記密碼？** 按鈕，輸入信箱後寄送含有 GUID 的超連結信件，導至密碼重設頁面
- 六碼圖形驗證碼(不須區分大小寫)

※ 說明語言：
1. 各欄位都有設驗證
2. 密碼有顯示/隱藏切換按鈕
3. 一律 Session Cookie：關瀏覽器即失效需重登（不提供「記住登入」功能，資安考量）；ExpireTimeSpan 24 小時純粹當 cookie 外洩防護的安全網
4. **忘記密碼?** 打開介面輸入信箱，利用SMTP寄送帶有GUID的連結至填入的信箱，點按此連結會導到 [重設密碼] 頁面

## 共通區域(頂部導覽列)
- 網站LOGO圖片與字樣，連結至首頁
- 梯次下拉選單選擇器
- 提醒鈴鐺(後續追加功能：留言通知使用)
- 角色頭像與名稱
- 個人資料
- 登出系統
- 一顆懸浮球，點按會展開字體控制按鈕以及留言通知按鈕(未來追加功能)

※ 說明語言：
1. 頂部導覽列友網站LOGO與名稱，為超連結，點按能回到首頁
2. 梯次下拉選單，能看到 ICON - 年度 - 梯次名稱 - 產學計畫區間日期，分類：進行中(包含準備中) / 已結案(歷史專案)
3. 有一個小鈴鐺按鈕，點按會開啟留言聊天室，會跳出有新留言的提醒
4. 有名稱的頭像與姓名，滑鼠移入會有 Hover 浮起效果，點按後會下拉展開 [個人資料] 與 [登出系統] 按鈕
5. 點按 [個人資料] 則會打開個人資料的蓋板區塊
6. 點按 [登出系統] 則會進行登出並清除 Session 或 Cookie

## 首頁
- 顯右上方顯示今日日期與時間
- 左半邊為頁面區塊卡片
    - 命題儀錶板：監控各題型缺口與整體命題進度。
    - 命題專案管理：梯次設定、派發工作與階段時程管控。
    - 命題總覽：全梯次試題列表、題目內容與審題全局結果檢視。
    - 命題任務：我的命題作業、審修任務與審核結果歷程。
    - 審題任務：互審 / 專審 / 總召審題的決策、退回與歷史。
    - 教師管理系統：教師人才庫、跨梯次命審歷程與帳號管理。
    - 角色與權限管理：系統帳號新增、外部人員身份指派及功能開關。
    - 系統公告/使用說明：管理內部看板公告及操作手冊下載。
- 右半邊為使用說明手冊下載、提醒看板與公告看板

※ 說明語言：
1. 右上方有今天的日期與時間
2. 左半邊為頁面連結的區塊按鈕，會根據導覽列梯次底下使用者的權限顯示對應的區塊(若不可使用，網址偷渡會被踢回首頁)
3. 右半邊為使用手冊說明以及警示提醒和公告看板
4. 警示提醒會在梯次階段剩餘 5 天轉換階段前計算未完成工作的人員名單與缺額
5. 公告與通知會根據公告的設定(全站公告、綁定梯次公告)與上下架時間來顯示應該顯示的公告

## 命題儀表版
- 上方有 [專案設定]、[試題總覽]按鈕可以連結至對應頁面
- 內容資料都是對應導覽列選擇的梯次
- 共有三列卡片
    - 第一列計數卡片：總目標數、已納入題庫(採用)、各審題階段、各修題階段
        - 總目標題數記錄專案建立時設定的題目數量
        - 已納入題庫(採用)在結案時才會計算資料，底下有採用率的補條
        - 各審題階段，在互審、專審、總審階段時會顯示 XX/OO 題。XX = 目前完成題、OO = 總題數。底下為現在階段與時間
        - 各修題修段，在互修、專修、總修階段時會顯示 XX/OO 題。XX = 目前完成題、OO = 總題數。底下為現在階段與時間
    - 第二列圖表：題型缺口達成率、依題型依題型狀態分佈表
        - 題型缺口達成率顯示七種提醒的 目前題數/目標題數
        - 依題型狀態分佈表，根據目前所有已命題目狀況顯示對應內容
    - 第三列通知： 
        - 提醒逾期或緊急代辦、最新歷程LOG

## 命題專案管理
- 頁面採左右分割版面（左側列表 1/3、右側詳情 2/3）
- 右上方按鈕：[新增梯次]
- **左側：專案列表**
    - 搜尋框：搜尋專案名稱 / 年度 / 合作學校
    - 狀態篩選按鈕列：全部 / 準備中 / 進行中 / 已結案
    - 列表項目：專案名稱、狀態徽章、年度、合作學校（無則顯示「無 (自辦)」）
    - 列表為空時顯示對應空狀態（從未建立過 → 引導建立第一個梯次；搜尋無結果 → 提示調整關鍵字）
- **右側：專案詳情**
    - 未選擇時顯示空狀態提示
    - 頂部資訊卡：狀態徽章、年度、專案名稱、合作學校、建立者，以及操作按鈕 [編輯專案]、[結案入庫]、[移除專案]
    - 過期未結案警示：產學區間結束日已過但尚未結案時，顯示赤陶色警告條提醒管理員結案
    - 時程規劃設定：以時間軸節點呈現 8 個階段的起迄日期，當前進行中階段以 morandi 藍色強調；已結案專案則用 terracotta 標出結案發生在哪一個子階段
    - 題型目標數量：3 欄 grid 顯示七種題型的目標題數
    - 指派人員：成員清單，每筆顯示頭像、姓名、教師編號，右側並列其角色徽章
- **新增 / 編輯 SlideOver 表單**（分三區塊）：
    1. 基本設定：所屬年度（由產學區間自動帶入）、專案名稱、合作學校（選填）
    2. 時程規劃設定：8 階段日期（產學區間 / 命題 / 互審 / 互修 / 專審 / 專修 / 總審 / 總修）
        - 設定第一階段起始日後自動推算半年內全部階段
        - 任一階段結束日修改會自動將下一階段開始日推進為隔天
    3. 目標題數與人員指派：
        - 七種題型題數欄位（題組類以「組」為單位）
        - 人員指派列表（資料來源為教師管理系統人才庫），每位人員可勾選並指派一個主要身份 + 多個附加身份
        - 命題階段結束後，題型題數與人員指派都會自動鎖定（不可再修改）
- 多人線上同步：透過 SignalR `ProjectsHub`，任一管理員的修改會即時推播給其他連線者

※ 說明語言：
1. 此頁專供管理員與計畫主持人使用，外部教師看不到
2. 整頁聚焦在左側列表 + 右側詳情的閱覽體驗
3. 新增梯次時只需設定第一階段「產學計畫區間」的開始日，系統會自動推算後續 7 個階段
4. 任一階段結束日改動，下一階段開始日會自動連動推進為隔天，可手動覆寫
5. 人員指派可給同一位教師多個身份（例：命題教師 + 互審），主要身份 + 附加身份用 + / × 按鈕新增刪除
6. 一旦命題階段結束（進入審題流程），題型題數與人員指派會鎖定灰色，不能再變更
7. 教師清單從 [教師管理系統] 的人才庫帶入，不能在此直接新增老師
8. [結案入庫] 後該梯次的所有題目鎖定，僅「採用」狀態入庫，其餘判定為不採用

## 命題總覽
- 上方統計卡片（5 張）：
    - 總題數
    - 命題進行中（草稿 + 命題完成）
    - 已採用（含結案入庫）
    - 待修編
    - 各審查環節：一張卡內以三聯格顯示「互審 / 專審 / 總審」當前梯次階段對應的待審 + 已給意見題數，非當前階段的格灰掉顯示 0
- 篩選工具列（4 個欄位 + 右側題數）：
    - 關鍵字搜尋：題號 / 題幹 / 命題教師
    - 題型下拉
    - 狀態下拉：依梯次實際存在的狀態動態渲染（optgroup 分組），無資料的選項自動省略
    - 命題教師下拉：僅列此專案有命題紀錄的老師（顯示題數）
    - 右側顯示「共 N 題」
- 試題列表（12 欄 grid，水平捲動 min-w-[1080px]）：
    - 題號（含最後修改時間 + 子題數 badge）
    - 命題教師
    - 題型
    - 等級 / 難度（合併同一欄，上下分行排列）
    - 三審狀態軌跡（7 階段燈號，由 `PhaseProgressStepper` 元件呈現，最小寬度 480px）
    - 當前狀態 Badge
- 七階段燈號順序：命題 → 互審 → 互修 → 專審 → 專修 → 總審 → 總修
- 「當前狀態」Badge 採四維度判定（題目 Status × 梯次 PhaseCode × 本人是否送出修題 × 全體審題者是否已給意見），會把命題端的「已送審 / 互審中 / 專審中」等用詞改寫成管理員視角的：
    - 「未完成命題」（紅）：草稿落隊（梯次已進入交互審題後仍是 Draft）
    - 「OO 完成」（莫蘭迪藍 inline badge）：題目已進入修題狀態但梯次還停在前一階段
    - 「修題已送出」（綠）：本人於本階段已送出修題說明
    - 「待審」（橘）/「已給意見」（綠）：審題階段依全體審題者是否已給意見切換
    - 「命題刪除」（danger）：已軟刪除題目，旁邊提供 [復原] 按鈕
- 點擊任一題列開啟右側 Slide-over 詳情（max-w-3xl）：
    - 上方：題目完整內容（依題型套用對應的 `QuestionPreviews/*` 元件，含選項、文章、音檔）
    - 下方：審題意見歷程（`ReviewHistoryTimeline` 元件，管理員視角不匿名、顯示真實姓名）
    - ESC 鍵可關閉
- 唯讀模式，不能在此頁修改題目；唯一寫入動作為「復原已刪除題目」
- 從命題儀表板教師卡片跳轉時會帶 `?creatorId=X` query string，自動套用該教師為命題教師篩選

※ 說明語言：
1. 此頁供管理員、計畫主持人、總召、教務管理者監看整體命題進度，外部教師看不到
2. 沒有等級、難度、目前階段這幾個獨立篩選；篩選只有關鍵字、題型、狀態、命題教師
3. 「狀態」下拉是依當前梯次實際存在的題目狀態動態產生，避免使用者選了但篩到 0 筆
4. 「當前狀態」Badge 文字與命題端不同，是 Overview 專屬的管理員視角語意
5. 詳情面板「題目內容」在上、「審題歷程」在下，方便對照
6. 此頁所有資料依導覽列選擇的梯次過濾

## 命題任務
- 三大頁籤：[命題作業區]、[審修作業區]、[審核結果與歷史]
- 上方命題配額進度條：依七種題型顯示「目前題數 / 目標題數」與完成率
- 各頁籤對應統計卡片（點擊卡片即套用對應狀態的快速篩選）：
    - 命題作業區（4 張）：命題總計、命題草稿、命題完成、已送審
    - 審修作業區（4 張）：審題總計、審題鎖定、修題中、已修題
    - 審核結果與歷史（3 張）：全部題目、已採用、不採用
- 篩選工具列（3 個欄位 + 右側題數）：
    - 關鍵字搜尋（題號或關鍵字）
    - 題型下拉
    - 等級下拉
    - 右側顯示「共 N 題」
    - **沒有獨立的「狀態」下拉**；狀態過濾是透過點擊上方統計卡片來切換
- 題目列表欄位（7 欄）：題號、題型、等級 / 難度（合併同欄）、題目摘要（含子題數 badge）、狀態、最後修改、操作
- 操作按鈕依狀態動態變化：
    - 草稿：[編輯]、[刪除]（命題階段未結束才出現）
    - 命題完成：[編輯]、[命題送審]
    - 已送審 / 審題鎖定：[檢視]
    - 修題中：[進入修題]
    - 審核結果歷史：[檢視詳情]
- **[命題作業區] 與 [審修作業區] 使用兩套完全不同的表單設計（兩個 Modal 互相獨立）**：

    **① 命題 Modal（[命題作業區] 專用，FullScreen）— 「專注命題型」設計**
    - 設計原則：盡量減少畫面干擾，讓教師專注在「把題目寫好」這件事
    - 兩欄結構：
        - 左側可收合「屬性側邊欄」：題型、等級、難度、主類 / 次類、文體、語音類型、素材分類、核心能力、細目指標
        - 右側為對應題型表單元件
    - Header 只有 [預覽] + 可編輯時的 [存為草稿] / [命題完成] 兩顆主按鈕
    - 標題列只放題目最後修改時間與狀態 Badge
    - **不顯示任何審題意見、歷程、決策資訊**（純粹創作環境）
    - 檢視模式（已送審 / 鎖定）整個 fieldset disabled，所有欄位變灰
    - `CloseOnBackdropClick="false"`，背景點擊不會關閉

    **② 修題 Modal（[審修作業區] 專用，`RevisionSlideOver` 但實為 FullScreen）— 「全資訊型」設計**
    - 設計原則：與 [審題任務] 的審題 Modal 風格一致，把所有意見、歷程、決策資訊都放在同畫面
    - 三欄結構：
        - 左欄：屬性側邊欄（同命題 Modal）
        - 中欄：題目編輯區（命題教師依審題意見修改題目本體）
        - **右欄：修題參考粘性側欄**（核心差異），含三層摺疊區：
            - 審題意見：互審（morandi）/ 專審（terracotta）/ 總審（amber）分色摺疊卡，預設展開
            - 我的修題歷程：自己歷次的修題說明，sage 安心綠，預設摺疊
            - 本次修題說明：CTA 主角區（必填，sage 強化），命題教師於此送出本輪修題說明
    - Header 顯示階段 Badge（互修 / 專修 / 總修）+ 總審退回 ≥ 2 次的紅色警示徽章 + 最後編輯時間
- 七種題型對應元件：
    - 一般題目（Single）、精選題目（Select）— 共用 PreviewSingle / 表單邏輯
    - 長文題目（LongText）
    - 短文題組（ShortGroup，多子題手風琴；母題 Topic / Subtopic 寫死為 6 / 17）
    - 閱讀題組（ReadGroup，子題 ABCD 答案與屬性側邊欄雙向同步）
    - 聽力題目（Listen，難度連動核心能力與細目指標）
    - 聽力題組（ListenGroup，固定 2 子題，難度三 / 難度四，母題本身無等級）
- Quill 富文本三層架構（QuillEditorHost + QuillField + QuillEditor）：所有輸入框點擊後從畫面底部滑出共用編輯器，含中文標點快插列
- 預覽功能：點擊 [預覽] 開啟模擬考卷外觀的 Modal，依題型套用對應 `QuestionPreviews/*` 元件

※ 說明語言：
1. 此頁為命題教師最常使用的頁面，依目前梯次階段顯示對應題目
2. 命題草稿可隨意編修，命題送審後鎖定不能改
3. 修題階段（互修 / 專修 / 總修）開放後，題目自動進入 [審修作業區] 並顯示 [進入修題] 按鈕
4. 命題與修題 Modal 都採用 **FullScreen 全螢幕**，背景點擊不會關閉，必須用左上 [返回列表] 或 ESC
5. 配額計算的「已完成」涵蓋 Status ≥ 1（命題完成、已送審、各審題與修題、採用），不只 `Completed=1`
6. 命題階段一旦結束，命題作業區進入鎖死狀態：不能新增 / 編輯 / 刪除草稿、配額不再計算、按鈕剩下 [檢視]
7. 同一頁面同時只會有一個 Quill 編輯器實體，所有輸入框共用
8. 七種題型的表單欄位、屬性側邊欄、能力指標完全獨立，符合中檢規範

## 審題任務
- 兩大頁籤：[審題作業區]、[審核結果與歷史]
- **[審題作業區] 頂部專屬：七階段時程燈號條**
    - 依 MT_ProjectPhases 的 7 個階段呈現燈號（done / active / closed / upcoming）
    - 燈號下方一行文字：目前階段名稱 + 剩餘天數（剩 5 天內變紅）；專案結案時改顯示結案日與結案階段
- 統計區域（兩個 Tab 顯示方式不同）：
    - **[審題作業區] — 沒有統計卡片**，只有一條 inline 統計摘要 bar：「本區題目」、「待處理」、「已審題」、「歷史紀錄」（>0 才顯示）
    - **[審核結果與歷史] — 3 張統計卡片**：全部題目、已採用、不採用
    - 命題階段（PhaseCode=2）審題作業區整條摘要 bar 隱藏，改顯示「等待進入審題流程」提示
- 篩選工具列（命題階段時整條隱藏；4 個欄位 + 右側題數）：
    - 關鍵字搜尋（題號或關鍵字）
    - 題型下拉
    - 等級下拉
    - 狀態下拉（內容隨 Tab 變化）：
        - 審題作業區：所有狀態 / 待審 / 已審題 / 歷史紀錄
        - 審核結果與歷史：所有狀態 / 已採用 / 不採用
    - 右側「共 N 題」
- 審題作業區列表欄位（8 欄）：題號、題型、等級（含難度）、題目摘要、**階段**（互審 / 專審 / 總審 分色 Badge）、**狀態**、編輯時間、操作
    - 狀態 Badge 文字：待審（amber）/ 已給意見（互審獨有）/ 已審：採用 / 已審：改後採用 / 已審：不採用 / 已退回（PhaseCode=8 的 Revise / Reject）/ 歷史紀錄（鎖頭灰）
- 審核結果與歷史列表欄位（7 欄）：題號、題型、等級、題目摘要、最終狀態、決策時間、操作
- 操作按鈕決策矩陣（審題作業區）：
    - 歷史紀錄（先前階段已結束）：[檢視] 灰
    - 待審：[審題] 綠（開啟 ReviewModal 進行決策）
    - 已決策（Completed）+ 總審第 3 次解鎖（`CanFinalReviewerEdit && Stage=Final && PhaseCode=8`）：[編輯題目] 橘 — 由總召親自修題
    - 已決策（Completed）其他情況：[檢視] 灰（題目已退回，等命題老師改）
- 操作按鈕（審核結果與歷史）：一律 [檢視]
- 審題 Modal（`ReviewModal`）：
    - 題目完整內容區（套用對應 `QuestionPreviews/*` 元件）
    - `ReviewSimilarityBanner`：模擬試題重複比對橫幅
    - `ReviewDecisionBar`：依階段顯示 5 種配置之一（互審無決策按鈕；專審只有採用 / 改後再審；總審完整三選項；總審第 3 次解鎖總召自改；唯讀檢視）
    - `ReviewActionPanel` + `ReviewQuestionDisplay`：審題人填寫意見（Quill 編輯器 + 8 則罐頭訊息 JS 快插，意見字數 `MinCommentLength=10`）
    - `ReviewHistoryTimeline`：歷次審題意見時間軸（Reviews 頁面為匿名顯示，「不採用」紅色卡片特殊樣式）
    - 總審退回計數 ≥ 2 時，Modal 頂部顯示「已退回 2 次」紅色預警徽章
- 三審制度與迴避規則：
    - 互審：命題教師互審，**只能給意見**（無採用 / 退回按鈕），自己命的題目不分配給自己
    - 專審：專家學者，可「採用」或「改後再審」（**無不採用**），自己命的題目不分配給自己
    - 總審：可「採用」、「改後再審」、「不採用」；**最多退回 2 次**，第 3 次由總審親自修並下最終裁決
    - 總召同時為專審委員時，總審階段不分配其在專審階段已審過的題目（自動轉派其他總召）
- 結案時：僅「採用」入題庫，其餘一律不採用

※ 說明語言：
1. 此頁是命題教師、審題委員、總召共用的頁面，依角色與梯次階段顯示對應任務
2. 審題作業區沒有統計卡片，只有一條 inline 摘要 bar；統計卡片只有審核結果與歷史 Tab 才有
3. 互審階段純粹是意見交流，沒有採用 / 退回的決策權
4. 專審階段開始可以決策，但沒有「不採用」這種重判
5. 總審階段擁有最高權限，可採用、退回、不採用；退回 2 次後第 3 次解鎖 [編輯題目] 由總召親自修
6. 退回的題目會回到命題教師端的 [審修作業區] 等候修改後重新送審
7. 一旦總審判定為「採用」，題目直接進入 [審核結果與歷史]，後續不再開放修改
8. 結案時，所有非「採用」狀態都自動歸為不採用，不入題庫

## 教師管理系統
- 頁面採左右分割版面（左側列表 380~420px、右側詳情）
- 上方統計卡片（4 張）：人才庫總數、帳號啟用中、帳號停用中、本梯次參與教師
- 右上方按鈕：[匯出名單]、[新增教師]
- **左側：教師列表**
    - 搜尋框：姓名 / 編號 / 學校 / 信箱
    - 狀態篩選按鈕：全部 / 啟用中 / 停用中
    - 列表項目：頭像（首字 + 啟用狀態圓點）、姓名、TeacherCode、學校、角色 Badge（內部紫 / 外部琥珀，未參與本梯次顯示「未參與此梯次」灰標）
- **右側：教師詳情**
    - 頂部資訊卡：72×72 大頭像、姓名 + 啟用狀態徽章、`ID: TeacherCode`、Email；右側垂直堆疊三個操作按鈕：[編輯資料]、[停用帳號 / 啟用帳號]、[重設密碼]
    - 詳情區共 **4 個分頁**：
        1. **[基本資料]**（info）— 包含三個子區塊：
            - 個人資料：姓名、性別、電子信箱、聯絡電話、身分證字號（遮罩顯示）、帳號建立日、首次登入狀態、最後登入時間
            - 任教背景：任教學校、系所 / 科別、職稱、專長領域、教學年資、最高學歷
            - 備註（有填寫才顯示）
        2. **[命題歷程]**（compose）— 4 張統計小卡（累計產出 / 已採用 / 不採用 / 審查中）+ 跨專案篩選下拉 + 命題列表（題號、題型、等級、所屬專案、狀態 Badge）
        3. **[審題歷程]**（review）— 3 張統計小卡（審題總數 / 已完成 / 待審中）+ 跨專案篩選下拉 + 審題列表（題號、題型、等級、所屬專案、**階段 Badge** + **決策 Badge** 雙標籤）
        4. **[參與專案]**（projects）— 右上角 [加入梯次] 按鈕；以卡片清單呈現每個梯次：
            - 狀態徽章（進行中 / 準備中 / 已結案）+ 專案編號
            - 年度 + 專案名稱
            - 角色徽章列（內部紫 / 外部琥珀）
            - 統計列：命題 N 題、採用 N 題、採用率 N%
            - 非結案梯次顯示 [移除] 按鈕
- **新增 / 編輯 SlideOver 表單**：分三區塊（基本資料、任教背景、帳號設定）
- **[加入梯次] Modal**：選擇專案 + 勾選身分（命題教師 / 互審 / 專審 / 總召）
- **[重設密碼]**：將密碼重置為預設 `CSF@01024304`（公司統編），首次登入強制變更
- TeacherCode 格式：`T` + 民國年 + 3 碼流水號（程式自動產生）

※ 說明語言：
1. 此頁專供管理員 / 計畫主持人使用，管理外部教師人才庫
2. 詳情面板四個分頁分別是 **基本資料 / 命題歷程 / 審題歷程 / 參與專案**；任教背景不是獨立分頁，而是基本資料分頁裡的子區塊
3. 教師帳號以信箱登入，預設密碼為 `CSF@01024304`，首次登入強制改密碼
4. 命題與審題歷程都有獨立的「跨專案篩選下拉」，可即時切換看單一專案或全部
5. 審題歷程列表會同時顯示「階段」和「決策」兩個 Badge（互審只給意見、專審 / 總審才有採用 / 改後再審 / 不採用決策）
6. [加入梯次] 等同於把該老師寫入該專案的人員清單，與 [命題專案管理] 頁面的人員指派雙向同步
7. 已結案的梯次不能移除教師

※ 說明語言：
1. 此頁專供管理員 / 計畫主持人使用，管理外部教師人才庫
2. 教師帳號以信箱登入，預設密碼為 `CSF@01024304`（公司統編），首次登入強制改密碼
3. 每位教師都會留下跨梯次的命題與審題歷程，方便後續評估與派任
4. 加入梯次的同時可指派身分，相當於同步更新該專案的人員清單
5. 已結案的專案無法移除教師
6. 教師管理與 [角色與權限管理] 為兩個獨立入口：此處管外部教師、Roles 管內部同仁

## 角色與權限管理
- 兩大頁籤：[人員帳號管理]、[角色權限管理]
- 右上方按鈕為**動態按鈕區**，依當前 Tab 切換：accounts → [新增人員] / roles → [新增角色]

### Tab 1：人員帳號管理（管理內部 CWT 同仁）
- 統計卡片（4 張）：帳號總數、啟用中、停用中、系統管理員
- 左右分割版面（左 380~420px / 右）：
    - **左側：帳號列表**
        - 搜尋框（姓名 / 帳號 / 信箱）
        - 狀態篩選按鈕：全部 / 啟用中 / 停用中
        - 列表項目：頭像（首字 + 啟用狀態圓點）、姓名、角色標籤、Username、公司職稱
    - **右側：帳號詳情**
        - 頂部資訊卡：頭像、姓名 + 啟用狀態 Badge、`帳號：Username`、Email；右側垂直堆疊三顆操作按鈕：[編輯資料]、[停用帳號 / 啟用帳號]、[重設密碼]
        - 詳情雙欄卡：
            - **基本資料**：姓名、登入帳號、電子信箱、公司職稱、建立日期、首次登入、備註
            - **權限資訊**：身份別（角色名 + 預設角色標籤）、角色分類（內部 / 外部）、可存取功能區塊 Badge 列表
- [新增人員 / 編輯人員] 開啟 SlideOver 表單
- 預設密碼為公司統編 `01024304`，首次登入強制變更

### Tab 2：角色權限管理（同時管內、外部所有角色）
- 統計卡片（**3 張**）：角色總數、內部人員角色、外部人員角色
- 角色卡片 Grid（1 / 2 / 3 欄響應式），每張卡片：
    - 圖示（內部紫 / 外部琥珀）+ 角色名稱
    - **預設角色顯示鎖頭**（命題教師、審題委員、總召）
    - 分類標籤（內部人員 / 外部人員）
    - 已啟用功能 / 總功能 計數（如 `5/8`）
    - 角色描述
    - 已啟用功能 Badge（最多顯示 6 個，超過顯示 `+N`）
    - Footer：
        - 左：`N 位使用者`（>0 時可點擊開啟「角色使用者清單 Modal」）
        - 右：自訂角色顯示 [編輯] + [刪除]，預設角色僅顯示 [檢視]
- **角色 Modal**（Center 對話框）：
    - 角色名稱、分類、描述
    - **功能模組 Toggle 列表**：依 `MT_Modules` 資料表動態載入（**不是寫死 8 個**）
    - **每個模組就是單純的 ON / OFF 開關**，沒有「僅瀏覽 / 瀏覽與編輯」兩級權限
    - 預設角色進入時所有 Toggle 灰色不可動，標題顯示「檢視角色」
- **角色使用者清單 Modal**：列出所有正在使用此角色的人員，含 Source 欄位區分「系統角色」（MT_Users.RoleId）或「梯次指派」（MT_ProjectMemberRoles）

※ 說明語言：
1. 人員帳號管理是管理 CWT 內部同仁，帳號為自訂帳號（如 `admin`），不是信箱
2. 角色權限管理同時涵蓋內部與外部身分（外部身分如命題教師、審題委員會被指派給外部教師）
3. 預設角色（命題教師、審題委員、總召）的權限由系統鎖定，自訂角色才能編輯刪除
4. 功能模組數量是動態的，依 `MT_Modules` 資料表載入；目前共 8 個模組對應 8 個頁面
5. 模組權限就是單純的開 / 關，沒有兩級分權；公告頁面也是統一一個開關
6. 外部教師（命題教師、審題委員）僅可進入命題任務、審題任務頁面，其餘禁止頁面網址偷渡會被踢回首頁
7. 自訂角色若有人正在使用，刪除會被擋下並提示

## 系統公告/使用說明
- 上方統計卡片（5 張）：公告總數、已發佈、草稿、已下架、置頂中
- 右上方按鈕：[新增公告]
- 篩選工具列（3 個欄位 + 右側題數）：
    - 關鍵字搜尋（公告標題或內容）
    - 分類下拉：所有分類 / 系統公告 / 命題公告 / 審題公告 / 其它
    - 狀態下拉：所有狀態 / 已發佈 / 未發佈 / 草稿 / 已下架
    - 右側「共 N 則公告」
- 公告列表（12 欄響應式 grid）：
    - 置頂（icon 或 — 占位）
    - 分類（彩色標籤）
    - 公告標題（含 TOP 紅標 + 內容預覽 60 字）
    - 綁定梯次（lg 才顯示，全站廣播 / 梯次名稱）
    - 狀態 Badge
    - 發佈日（md 才顯示）
    - 操作按鈕：[編輯]、[置頂 / 取消置頂]（僅已發佈 / 未發佈才有）、[刪除]

- **新增 / 編輯 SlideOver 表單**（分兩區塊）：
    - 區塊 1：公告設定
        - 分類下拉（系統公告 / 命題公告 / 審題公告 / 其它）

- 自動下架：超過下架日期前端自動推導 `DisplayStatus="已下架"`
- 點擊背景自動存草稿（`HandleBackdropSaveDraft`，需標題 + 內容皆有值）
- 自動取消置頂：草稿、已下架狀態勾置頂時前端會 Toast 警告並自動取消
- 排序規則：置頂 > 顯示狀態（已發佈 > 未發佈 > 草稿 > 已下架）> 分類順序 > 發佈日新到舊

1. 此頁與首頁的「公告看板」資料連動，所有發佈中的公告都會即時顯示在首頁右側
2. 全站廣播代表所有人都看得到，指定梯次則只有該梯次的相關人員看得到
3. 置頂的公告會顯示在首頁公告區的最上方紅色醒目區塊
4. 上下架日期讓公告自動依時間生效與失效，不需手動切換狀態
