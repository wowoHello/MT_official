# 命題任務頁面 — 功能與設計說明書

> **文件版本**：v1.1  
> **最後更新**：2026-03-04  
> **對應頁面**：命題任務（cwt-list.html）

---

## 一、這個頁面是做什麼的？

「命題任務」是整個 CWT 命題工作平臺裡 **最核心的頁面**。

命題教師在這裡完成所有跟出題有關的事情：新增題目、編輯題目、預覽考卷效果、把題目送審。

簡單來說，這就是老師們每天上線後待最久的其中一個地方（另一個是審題任務）。

---

## 二、畫面從上到下長什麼樣？

整個頁面由上而下大致分成這些區塊：

1. **頂部導覽列** — 左邊是平臺 Logo，中間可以切換不同梯次（專案），右邊顯示登入者姓名和登出按鈕
2. **麵包屑** — 顯示「首頁 > 命題任務」，讓使用者知道自己在哪
3. **頁面標題與說明** — 簡短描述這頁的用途
4. **命題配額進度** — 顯示被分配到的各種題型命題進度
5. **分頁標籤** — 分成「命題作業區」和「審修作業區」及「審核結果與歷史」三個頁籤
6. **篩選工具列** — 可以依題型、等級、狀態、關鍵字來篩選題目
7. **題目列表** — 表格呈現所有題目，可以勾選、預覽、編輯、刪除
8. **分頁器** — 翻頁用

另外還有兩個彈出視窗（不會一開始就看到）：
- **命題表單** — 全螢幕彈窗，用來新增或編輯一道題目
- **考試預覽** — 模擬考卷的感覺，讓老師看看題目印出來大概長什麼樣

畫面最底部還有一個隱藏的 **滑入式文字編輯器(Quill編輯器)**（類似手機打字的鍵盤），打開後可以輸入富文本內容。
此頁面所有的輸入框都需要綁此 Quill編輯器
---

## 三、各區塊功能說明

### 3.1 專案切換器

放在導覽列正中央。點下去會展開一個下拉面板，裡面列出「進行中」和「已結案」的梯次，還有搜尋框可以快速找到想要的梯次。切換梯次後，整個頁面的資料會跟著變動。

### 3.2 分頁標籤

| 標籤 | 用途 | 對應的統計卡片 |
|------|------|-------------|
| 命題作業區 | 管理正在編輯、即將送審的題目 | 草稿 / 完成 / 送審(進入審修作業區) |
| 審修作業區 | 管理OO修題的題目 | 審查鎖定 / 解鎖修題 |
| 審核結果與歷史 | 查看已進入最終判決的題目結果 | 採用 / 不採用 |

切換標籤後，下方的統計卡片、狀態篩選選項、以及表格列的顯示都會自動跟著調整。

> **與審題的關係**：命題教師送審後，題目進入審修作業區也就是「待審佇列」等待分配。各審題階段（交互審題/專家審題/總召審題）會鎖定命題編輯功能；對應修題階段（互審修題/專審修題/總審修題）會解鎖編輯功能，教師可依審題意見修改後再次送審。當題目被總審判定為「採用」或「不採用」後，才會進入「審核結果與歷史」Tab。

### 3.3 篩選工具

提供 4 種篩選方式，可以任意組合使用：

| 篩選方式 | 說明 |
|----------|------|
| 題型篩選 | 下拉選單，選擇 7 種題型之一或「全部」 |
| 等級篩選 | 下拉選單，選項會根據選的題型自動改變（例如聽力題會變成難度一～五） |
| 狀態篩選 | 下拉選單，依當前分頁標籤列出對應狀態 |
| 關鍵字搜尋 | 輸入文字後即時過濾，按題目摘要內容搜尋 |

### 3.4 統計卡片

每個TAB頁籤有各自的計數卡片，即時顯示各狀態的題目數量。

**命題作業區的 4 張卡片**：
- **命題總計** — 點擊可看全部；
- **命題草稿** — 點擊卡片可篩選草稿；
- **命題完成** — 點擊卡片可篩選完成；
- **已送審** — 點擊卡片可篩選已送審；

**審修作業區的 3 張卡片**：
- **審題總計** — 點擊卡片可看全部；
- **審題鎖定** — 點擊卡片可篩選審查鎖定的題目；
- **修題中** — 點擊卡片可篩選能夠修題的題目；

**審核結果與歷史的 3 張卡片**：
- **全部題目** — 點擊卡片可看全部；
- **已採用** — 點擊卡片可篩選已採用；（僅篩選用，題目經總審判決後才會出現在此）
- **不採用** — 點擊卡片可篩選不採用；（僅篩選用，題目經總審判決後才會出現在此）

### 3.5 題目列表

表格呈現每道題目的基本資訊：

| 欄位 | 說明 |
|------|------|
| 題目編號 | 顯示這道題的編號 |
| 等級 | 小標籤呈現（初級、中級…） |
| 難度 | 易/中/難 |
| 題型 | 一般題目、長文題目 等文字 |
| 題目內容摘要 | 顯示這道題的精簡描述 |
| 狀態 | 彩色標籤，不同狀態不同顏色 |
| 建立時間 | 首次建立的時間 |
| 最後修改時間 | 最近一次修改的時間 |
| 操作 | 動態按鈕，依狀態不同而不同（見下方） |

**命題作業區的操作按鈕規則**：
- **命題草稿** → 右邊按鈕「編輯 | 刪除」
  - 編輯：打開Model內，右上有［預覽］［存為草稿］［命題完成］
  - 刪除：軟刪除，僅刪除頁面上資料，資料庫內資料會保留，所以題目的資料表需要一個狀態來紀錄是否被老師刪除。
- **命題完成** → 右邊操作按鈕「編輯 | 命題送審」
  - 編輯：打開Model內，右上有［預覽］［命題送審］
  - 命題送審：狀態改為**已送審**進入待審佇列。
- **已送審**（被鎖定的題目）→ 右邊按鈕只顯示「檢視」，不能改也不能刪（等待審題分配）
  - 檢視：打開Model內，右上有［預覽］

**審修作業區的操作按鈕規則**：
- **鎖定審查中** → 右邊按鈕「檢視」
  - 檢視：打開Model內，右上有［預覽］，底下的**歷程軌跡**內容可以看到每一階段的意見
- **修題中** → 右邊按鈕「進入修題」
  - 進入修題：打開Model內，右上有［預覽］［存為草稿］［完成修題］

- **審核結果與歷史中的題目**（採用、不採用）→ 只顯示「檢視詳情」，不能改也不能刪

**列的排序優先順序**：
1. 命題作業區：
  - 命題草稿 → 命題完成 → 根據編輯時間先後順序
2. 審修作業區：
  - 修題草稿 → 需修題 → 鎖定審查中 → 根據編輯時間先後順序

## 四、命題表單（彈出視窗）

這是一個佔滿整個螢幕的大彈窗，用來新增或編輯題目。

### 4.1 頂部共用欄位
不管選什麼題型，上方**Model Title**都會有這些欄位：
| 欄位 | 說明 |
|------|------|
| 返回列表 | 關閉 Model 的按鈕 |
| 開啟 Model 的狀態 | 編輯試題 / 檢視試題 |
| 對應題型的標籤 | 顯示各種題型對應的標籤項目 |
| 最後修訂日 | 自動記錄，不可修改 |
| 目前狀態 | 用彩色標籤顯示這道題當前的狀態 |
| 對應狀態的按鈕 | 預覽、存為草稿、命題完成等...按鈕 |

不管選什麼題型，左側**題目屬性**都會有這些欄位：

| 欄位 | 說明 |
|------|------|
| 題型選擇 | 下拉選單，切換後下方表單內容會跟著變 |
| 適用等級 | 下拉選單，選項會根據題型自動調整 |
| 對應題型的標籤 | 顯示各種題型對應的標籤項目 |
| 最後修訂日 | 自動記錄，不可修改 |
| 目前狀態 | 用彩色標籤顯示這道題當前的狀態 |

### 4.2 七種題型表單

每種題型的表單都是 **左側邊欄 + 右側主要內容** 的分欄排版。

---

#### ① 一般題目 / 精選題目（共用同一套表單）

**左側邊欄**：
- 主類（7 個分類可選：文字、語詞、造句標點…）
- 次類（依主類自動帶出對應選項）
- 難度（三選一按鈕：易 / 中 / 難）
- 正確答案（選 A/B/C/D，選完後右邊選項會出現綠色邊框標示）

**右側內容**：
- 題幹（點擊打開底部文字編輯器輸入，可附圖）
- 四個選項卡片（A/B/C/D，各自可輸入內容和附圖）
- 試題解析（說明為什麼選這個答案，無高度限制）

---

#### ② 長文題目

**左側邊欄**：
- 題型（引導寫作 / 資訊整合）
- 難度（易 / 中 / 難）

**右側內容**：
- 題目標題
- 文章內容（可附圖）
- 批閱說明

---

#### ③ 短文題組

**左側邊欄**：
- 主類／次類（固定為「文義判讀 / 篇章辨析」，不可改）
- 文體（文言文 / 應用文 / 語體文）
- 難度（易 / 中 / 難）

**右側內容**：
- **母題區**：題目標題 + 文章內容（可附圖）
- **子題區**：
  - 可以按「新增子題」按鈕，動態加入多道子題
  - 每道子題是一個可展開摺疊的手風琴卡片
  - 每道子題包含：題目內容、主向度選擇、能力指標選擇（會依主向度自動帶出）、試題解析
  - 子題填完後，手風琴標題會出現綠色勾勾✅

---

#### ④ 閱讀題組

**左側邊欄**：
- 文體（文言文 / 應用文 / 語體文）
- 難度（易 / 中 / 難）
- 正確答案下拉選單（跟目前展開的子題**雙向同步**）
  - 左邊改答案 → 右邊跟著變
  - 右邊改答案 → 左邊跟著變

**右側內容**：
- **母題區**：標題 + 文章內容（可附圖）
- **子題區**：
  - 可動態新增/刪除子題
  - 每道子題是手風琴卡片，含：題目 + ABCD 選項（可附圖）+ 正確答案下拉

---

#### ⑤ 聽力題目

**左側邊欄**：
- 核心能力 / 細目指標（依等級自動帶入，不可修改）
- 語音類型（對話 / 情境 / 陳述）
- 素材分類（生活 / 教育 / 職場 / 專業）
- 正確答案（A/B/C/D，選完右邊選項卡會標綠色邊框）

**右側內容**：
- 題目
- 語音內容（可上傳音檔）
- ABCD 四個選項卡片
- 試題解析

**特殊連動**：等級改變時，核心能力和細目指標會自動切換對應內容。例如選「難度一」→ 自動帶入「提取訊息 / 提取對話與訊息主旨」。

---

#### ⑥ 聽力題組

**左側邊欄**：
- 語音類型（對話 / 情境 / 陳述）
- 素材分類（生活 / 教育 / 職場 / 專業）
- 正確答案（跟展開中的子題雙向同步）

**右側內容**：
- **母題**：語音內容（可上傳音檔）
- **子題列表**：固定 **只有 2 題**，不能新增也不能刪除
  - 第 1 題：難度三 / 推斷訊息
  - 第 2 題：難度四 / 歸納分析
  - 每題的核心能力和指標已固定，各自含 ABCD 選項

**檢視模式（唯讀）**：如果是從「檢視」按鈕進來的，所有欄位都會變灰色不可編輯，頂部按鈕僅預覽。

## 五、共用元件

### 5.1 底部滑入式文字編輯器

這是整個頁面最有特色的設計。所有需要輸入文字的地方（題幹、選項、解析…），都不是直接在欄位裡打字，而是：

1. 點擊輸入框（看起來像一個灰色的方塊，裡面有「點擊輸入…」提示）
2. 畫面底部會像手機鍵盤一樣**滑上來**一個文字編輯器
3. 編輯完畢按「完成」，內容就會填回去
4. 所有輸入框共用同一個編輯器，不會開啟多個

**編輯器配備**：
- Quill 富文本編輯器（支援粗體、斜體、底線、字體大小等）
- 中文標點符號快捷列：`，` `。` `、` `？` `！` `：` `；` `「」` `『』` `（）` `【】` `……` 共 12 種
- 即時字數統計

### 5.2 考試預覽

點擊列表中的「預覽」按鈕後，會跳出一個模擬考卷外觀的彈窗，讓老師看看題目在考卷上大概長什麼樣子。

- 7 種題型各自有獨立的呈現方式
- 單選題會顯示 (A)(B)(C)(D) 標準考卷格式
- 題組會顯示文章 + 編號子題
- 聽力題會顯示一個音檔播放器的佔位圖示

## 七、狀態定義與流程

### 狀態說明

| 狀態 | 顏色 | 意義 | 可否編輯 | 所屬 Tab |
|------|------|------|---------|----------|
| 命題草稿 | ⚫ 灰色 | 老師正在編輯的題目 | ✅ | 命題作業區 |
| 命題完成 | 🔵 藍色 | 老師確認完成，但還沒送審 | ✅ | 命題作業區 |
| 命題送審 | 🟢 綠色 | 已提交至待審佇列，等待審題分配 | ❌（鎖定） | 命題作業區 |
| 鎖定審查中 | 🔵 藍色 | 已提交至待審佇列，等待審題分配 | ❌（鎖定） | 審修作業區 |
| 修題中 | 🟠 橙色 | 審題階段，修題時間內可修改 | ✅（修題期間） | 審修作業區 |
| 採用 | 🟢 綠色 | 總審判定通過 | ❌ | 審核結果與歷史 |
| 不採用 | ⚫ 灰色 | 總審最終判定未通過 / 結案時非「採用」 | ❌ | 審核結果與歷史 |

### 命題教師的操作流程

```
新增題目 → 命題草稿
    ↓ 確認完成
命題完成
    ↓ 單筆送審
命題送審（進入審修作業區鎖定狀態，進入待審佇列）
    ↓
┌─ 交互審題（互審老師給予意見，此階段無決策按鈕）
│   ↓ 互審修題（審修作業區解鎖，命題老師依意見修改後重新送審）
├─ 專家審題（專審老師審閱，可「採用」或「改後再審」，無不採用）
│   ↓ 專審修題（審修作業區解鎖編輯，命題老師依意見修改後重新送審）
└─ 總召審題（總審審閱，可「採用」「改後再審」「不採用」）
    ├── 採用 ✅ → 進入「審核結果與歷史」
    ├── 改後再審 → 總審修題（解鎖，老師修改後重新送審，最多退回兩次）
    │   └── 第三次不通過 → 總審可編輯，總審自行修改並做最終決策
    └── 不採用 ❌ → （功能同改後再審，此狀態是給命題老師知道該題問題很大）

※ 結案時：僅「採用」入庫，其餘一律判定為不採用
```

## 八、已知可改善項目

1. **資料存放方式不理想** — 目前所有題目資料是直接嵌在每一列的 HTML 屬性裡，未來遷移到 Blazor 後應改為從後端 API 取得資料，前端只負責顯示。

2. **七種題型的程式邏輯高度重複** — 每種題型都重複寫了「清空欄位」、「填入資料」、「收集資料」的邏輯。建議抽出共用的部分，各題型只需描述差異即可。減少將近 40% 的重複程式碼。

3. **全域函式過多** — 目前很多功能函式為了讓按鈕能直接呼叫而暴露在全域範圍。建議統一用事件委派的方式管理，減少全域污染。

4. **篩選效能可合併** — 目前篩選和統計計數各跑一次迴圈，可以合併成一次就好。

5. **Demo 資料應抽離** — 目前假資料直接寫在 HTML 裡面約 160 行。建議改由程式動態生成，這樣未來換成真的 API 回傳資料時，替換會更乾淨。
