# PRD: CWT 命題工作平臺

> **文件版本**：v1.3  
> **最後更新**：2026-03-11
> **維護者**：劉明杰 Jay

## 1. Introduction / Overview

**CWT 命題工作平臺**是一個為「全民中檢」設計的線上命題與審題管理系統。平臺提供管理員管理、教師命題、專家審題三大核心工作流程。使用者登入後依據帳號權限顯示對應的功能模組。

本專案目前為**前端 UI DEMO**，所有業務邏輯均以前端模擬方式實現（Mock Data + localStorage），不包含後端 API 與資料庫。預計未來遷移至 **Blazor .NET 10**。

### 目標使用者
- **命題教師**：負責命製試題並進行互審
- **審題委員 (專家學者)**：負責專審階段的題目審查
- **CWT 內部職員 (系統管理員)**：負責專案管理、總召審題、帳號與權限管理
- 使用者年齡層分佈：壯年至年長者（UI 須適合此年齡層閱讀，系統亦提供字體加大功能）
- **產學計畫負責人 (系統管理員)**：負責專案管理、總召審題、帳號與權限管理
- 使用者年齡層分佈：壯年至年長者（UI 須適合此年齡層閱讀，系統亦提供字體加大功能）

---

## 2. Goals

- 提供完整的命題與修題工作流程：七大階段（命題階段、交互審題、互審修題、專家審題、專審修題、總召審題、總召修題）
- 支援 7 種題型的獨立表單結構（一般、精選、長文、短文題組、閱讀題組、聽力、聽力題組）
- 實現三審制度（互審 → 專審 → 總審）與分歧裁決邏輯
- 提供命題專案（梯次）的生命週期與時程管理
- 提供角色與權限控管，依權限顯示對應功能模組（包含內部職員與外部人員）
- 提供命題總覽與儀表板以監控整體命題進度
- 提供系統公告與各階段的截止倒數提醒功能
- 提供教師人才庫管理功能
- 確保 UI/UX 適合壯年至年長者使用（字體可任意調整大小、配色柔和不刺眼）

### US-009: 審題任務 - 三審管控制度（關聯：reviews.html、cwt-ex-rules.md）
**Description:** 身為一位審題委員/管理人員，我希望能審閱分配給我的試題並給予意見或做出裁決。

**Acceptance Criteria:**
- [ ] 審題清單 Tab 切換：「審題作業區」（包含互審、專審、總審三個狀態，經決策過的會改為「已決策」）與「審核結果與歷史」（採用、不採用，僅能檢視）
- [ ] 審題作業區顯示 4 張統計卡片：待審總計、互審中、專審中、總審中
- [ ] 三階段審題制度與操作限制：
  - **互審（交互審題）**：命題教師之間互審，僅提供「給予意見」功能（**無** 採用、改後再審、不採用按鈕）。自己命的題目不分配給自己。
  - **專審（專家審題）**：專家學者審閱，提供「採用」與「改後再審」（**無** 不採用按鈕）。
  - **總審（總召審題）**：總召/負責人審閱，提供「採用」、「改後再審」、「不採用」。
- [ ] 退回修題限制：
  - 專審退回命題教師修改，送出後直接進入總審階段。
  - 總審退回可最多退回兩次，第三次再不通過不會退回教師，由總審直接修改並判定最終結果。
- [ ] 特殊迴避控制：
  - 教師可能同時為命題/審題人員，自己命題絕對不自審。
  - 總召（允許多人）若同時也為專審委員，進入總審階段時不可取得其在專審階段已審過的題目（將自動配給其他總審）。
  - 總召審題一旦判定為「採用」，試題直接進入「審核結果與歷史」，後續不再開放修題編輯。
- [ ] 審題 Modal 包含快速插入罐頭訊息及模擬的試題重複比對功能
- [ ] 總審「不採用」或梯次結案時未被採用的考題，皆視為不採用狀態進入歷史區
- [ ] Verify in browser using dev-browser skill

## 4. Functional Requirements

### FR-1: 登入考驗與提示
- 前端驗證包含 Canvas 形狀隨機噪點與干擾線。
- 「忘記密碼」新舊密碼比對防呆紅字提示。

### FR-2: 首頁導覽與字體控制
- 各頁面右下角常駐字體大小控制器球。首頁提醒看板必須過濾並置頂「倒數過期」的項目（紅字顯示）。

### FR-3: 專案（梯次）生命週期切換與同步
- 新增時包含 8 階段時程：產學區間、命題階段、交互審題、互審修題、專家審題、專審修題、總召審題、總召修題。
- 命題階段與各修題階段在倒數 5 天時，須在「今日提醒」觸發紅色倒數。
- 切換專案後，必須全站同步更新對應梯次的呈現資料。
- 「結案入庫」操作僅將最終狀態為「採用」者轉換為入庫有效題，其餘一律標記為不採用並鎖定。

### FR-4: 我的命題任務頁面機制
- 分別用三大 TAB：「命題作業區」、「審修作業區」、「審核結果與歷史」來過濾和管理各期狀態。
- 編輯視窗提供 7 大題型的表單與共同底層的 滑入式 Quill 編輯器（附繁體中文標點符號）。
- 新增與編輯狀態如未點擊「儲存」，點背景關閉視窗視同背景「存為草稿」。

### FR-5: 三審指派與迴避邏輯
- **自己命題不自審**：互審與專審皆強制隔離當前命題教師自己的題目。
- **總審前測隔離**：總召（不論幾人）參與總審時，扣除該總召曾在專審審過的題目，交給其餘總召或專員處理。
- **第一階與第二階決策限制**：互審無法給予決定，唯獨給意見；專審無法按下「不採用」。
- **兩次退回底線**：當總審退回達 2 次，滿第 3 次總審需自行修改，不再退回命題教師，由總審親自給予採用或不採用裁決。
- 按下審查決定後，審查作業區的按鈕狀態需替換成「已決策」。

### FR-6: 跨模組串接
- 命題專案指派人員選單內名單捕捉自「教師管理系統」人才庫。
- 首頁「今日提醒」連動至「命題專案管理」中設定的倒數階段期程。

## 5. Non-Goals (Out of Scope)

- 不包含真實後端 API 與資料庫儲存邏輯（完全透過前端模擬與 LocalStorage）。
- 不包含真實的寄送電子郵件或站內信通知。
- 不包含 AI 試題查重比對的真實運算（僅做 Modal UI 展示）。
- 不包含題庫系統的組卷與印卷模組（該模組為獨立子系統計畫）。
- 只支援單一語系（繁體中文），不製作 i18n 多國語言。

---

## 6. Design Considerations

### UI / UX 指引
- **色彩策略**：以 `#F0F2F5`（淺灰色）為底，柔和高對比度的配色，重點按鈕/導覽使用 `#2563EB`。
- **高階互動控制**：微弱陰影與 Hover 位移反饋（Card Hover 出現彩色邊框或浮動）。
- **長者親和性**：元件邊距應避免過於緊湊（依循 Tailwind CSS 版面設計），系統具備全局字體放大縮小按鈕。
- **前端防呆**：凡是非同步動作（如改變狀態提交）皆必須包含 Spinner 或禁用效果 `disabled` 防止重複提交。

## 7. Technical Considerations

- **HTML5, CSS3, ES6+ Vanilla JS** (完全禁止 jQuery, Lodash 等多餘外部框架)
- **Tailwind v4.2+** (核心排版與基礎 Component 提供)
- **Quill.js** (核心富文本引擎)
- **SweetAlert2** (核心對話框與 Alert 提示引擎)
- 未來遷移框架目標：Blazor .NET 10，保持高內聚與 Component 式思維建構純前端 JavaScript DOM 操作與資料分離。
