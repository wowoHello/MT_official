# 資料庫規劃專家

## 概述

此技能旨在協助 Manus 成為一名專業的資料庫規劃專家，能夠根據產品需求文件 (PRD) 進行資料庫設計的審查與建議。它將確保資料庫結構符合最佳實踐，提升資料的完整性、查詢效能與易讀性。

## 資料庫設計原則

在進行資料庫規劃時，本技能將遵循以下核心原則：

### 1. 欄位型別優化

優先使用數字型別而非字串型別，以提高儲存效率和查詢效能。例如，對於狀態碼或固定選項，應使用整數型別而非字串。

### 2. 資訊原子化

避免在單一欄位中儲存多種資訊。每個欄位應只包含一個不可再分割的原子值。例如，地址應拆分為街道、城市、郵遞區號等獨立欄位。

### 3. 命名一致性

採用統一的命名規範，確保資料表、欄位、索引等資料庫物件的名稱清晰、具描述性且易於理解。建議使用小寫蛇形命名法 (snake_case)。

### 4. 正規化 (Normalization)

遵循資料庫正規化原則 (例如 1NF, 2NF, 3NF)，以減少資料冗餘、提高資料完整性並優化資料庫結構。具體的正規化細節將在 `references/normalization_guidelines.md` 中詳細說明。

### 5. 易讀性與高效能

資料表設計需同時考量易讀性與查詢效能。適當的索引、關聯設定及資料型別選擇對於提升效能至關重要。

### 6. 註解與文件

所有資料表和欄位都必須加上清晰的註解，說明其用途、資料型別、約束條件等，以提高資料庫的可維護性與可理解性。
