# 命題系統單元測試技能 (MT Unit Test Skill)

## 概述

此技能旨在提供一個自動化的單元測試框架，用於驗證命題系統中不同使用者角色（系統管理員、命題教師、審題委員）登入後，其介面所顯示的功能模組是否符合預設的權限設定。這有助於確保系統的權限控制邏輯正確無誤，並提升開發與維護的效率。

## 快速開始

本技能的核心測試邏輯位於 `scripts/test_roles.js`。該腳本設計用於在瀏覽器環境中執行，特別是針對 `role-login-test.html` 頁面進行測試。若要執行測試，請按照以下步驟操作：

1.  **開啟測試頁面**：在瀏覽器中開啟您的命題系統專案中的 `role-login-test.html` 檔案。此頁面已內建模擬登入與模組渲染邏輯。
2.  **開啟開發者工具**：在瀏覽器中按下 `F12` 開啟開發者工具，並切換到 `Console` (控制台) 頁籤。
3.  **執行測試腳本**：將 `scripts/test_roles.js` 的內容複製並貼上到瀏覽器控制台中，然後按下 `Enter` 執行。測試結果將會直接顯示在控制台中。

## 測試案例說明

`test_roles.js` 腳本會依序模擬以下三種角色的登入，並驗證其登入後可見的功能模組：

### 1. 系統管理員 (Admin)

*   **預期行為**：登入後應能看到所有管理相關與命題相關的功能模組，包括儀表板、專案管理、總覽、我的命題任務、審題清單、教師管理、角色與權限管理、系統公告(管理)。
*   **驗證點**：檢查 `moduleGrid` 元素中是否包含所有預期的模組 ID，且沒有不屬於管理員的模組。

### 2. 命題教師 (Teacher)

*   **預期行為**：登入後應能看到與命題任務和審題相關的功能模組，以及系統公告的檢視權限，例如我的命題任務、審題清單、系統公告(檢視)。
*   **驗證點**：檢查 `moduleGrid` 元素中是否只包含命題教師應有的模組 ID，且不包含管理員專屬的模組。

### 3. 審題委員 (Reviewer)

*   **預期行為**：登入後應能看到與審題相關的功能模組，以及系統公告的檢視權限，例如審題清單、系統公告(檢視)。
*   **驗證點**：檢查 `moduleGrid` 元素中是否只包含審題委員應有的模組 ID，且不包含其他角色專屬的模組。

## 資源

本技能包含以下資源文件：

### `scripts/test_roles.js`

*   **功能**：包含實際的 JavaScript 單元測試邏輯，用於模擬不同角色的登入並驗證其 UI 模組顯示。
*   **使用方式**：直接在瀏覽器開發者工具的控制台中執行，或整合至自動化測試流程。

### `references/role_permissions.md`

*   **功能**：詳細列出系統中各角色的預期權限與對應的功能模組 ID，作為測試設計與驗證的參考依據。
*   **使用方式**：供開發者參考，以了解各角色的權限範圍。
