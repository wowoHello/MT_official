# PRD: 系統公告 / 使用說明（Announcements）

> **文件版本**：v1.0
> **最後更新**：2026-03-19
> **關聯檔案**：`announcements.html`、`js/announcements.js`、`js/shared.js`
> **目標遷移框架**：Blazor .NET 10

---

## 1. 頁面概述

系統公告 / 使用說明頁面為 CWT 命題工作平臺的資訊公佈中心，提供公告的 CRUD（新增、編輯、檢視、刪除）管理功能，以及使用說明手冊的存取入口。

公告可綁定特定梯次或全站廣播，支援置頂、自動下架、草稿管理等機制。公告內容連動至首頁「今日提醒」側邊欄的公告看板。

---

## 2. 使用者角色與存取權限

此頁面的操作權限由角色權限管理（PRD-Roles.md）中的 `announcements` 功能模組與 `announcementPerm` 層級控制。

| 權限層級 | 右上角按鈕 | 可執行操作 |
|---------|-----------|-----------|
| **瀏覽與編輯（edit）** | ［上傳使用說明手冊］＋［使用說明手冊］＋［＋ 新增公告］ | 全部操作（CRUD、置頂、上傳手冊） |
| **僅瀏覽（view）** | ［使用說明手冊］ | 僅可瀏覽公告列表與檢視內容、下載手冊 |

> **注意**：僅瀏覽權限下，列表操作欄（編輯、置頂、刪除按鈕）不顯示，行點擊僅開啟檢視 Modal，且檢視 Modal 底部的「編輯此公告」按鈕也隱藏。

## 4. 新增 / 編輯公告（Slide-over 面板）

從右側滑入的 Slide-over 面板，寬度 `max-w-3xl`。

### 4.1 面板標題

- 新增模式：「📢 新增公告」
- 編輯模式：「📢 編輯公告」

### 4.2 區塊 1：公告設定

| 欄位 | 類型 | 必填 | 預設值 | 說明 |
|------|------|:----:|-------|------|
| 分類 | select | ✅ | 系統公告 | 系統公告 / 命題公告 / 審題公告 / 其它 |
| 狀態 | select | ✅ | 草稿 | 草稿 / 已發佈 / 已下架 |
| 綁定顯示梯次 | select | | 全站廣播 | 「全站廣播 (所有梯次)」或選擇特定梯次；附提示文字說明差異 |
| 上架日期 | date | ✅ | 今日日期 | |
| 下架日期 | date | | 空（不自動下架） | 附提示：「留空表示不自動下架。超過此日期後，狀態將自動變為『已下架』。」 |
| 置頂公告 | checkbox | | 未勾選 | 紅底區塊，附說明：「開啟後此公告會在列表與首頁公告看板中優先顯示於最頂端。」 |

**綁定顯示梯次邏輯**：
- 「全站廣播」→ 所有梯次的使用者均可看到此公告
- 選擇特定梯次 → 僅在該梯次被選中時顯示（首頁提醒看板同理）
- 下拉選項動態注入自 `projectsRef`（命題專案管理的梯次清單）

### 4.3 區塊 2：公告內容

| 欄位 | 類型 | 必填 | 說明 |
|------|------|:----:|------|
| 公告標題 | text | ✅ | placeholder「請輸入公告標題」 |
| 公告內容 | Quill 編輯器 | ✅ | Rich Text，附中文標點快速插入列 |

**Quill 編輯器工具列配置**

| 工具群組 | 項目 |
|---------|------|
| 文字格式 | 字體大小（small / normal / large）、標題層級（H2 / H3 / 正文） |
| 樣式 | 文字顏色、背景色、對齊 |
| 格式化 | 粗體、底線、刪除線、超連結 |
| 列表 | 有序列表、無序列表、縮排增/減 |
| 媒體 | 插入圖片、清除格式 |

**中文標點快速插入列**

位於 Quill 工具列上方，提供以下標點按鈕：

| 按鈕 | 插入字元 | 配對模式 |

| ， | ， | |
| 。 | 。 | |
| 、 | 、 | |
| ； | ； | |
| ： | ： | |
| 「」 | 「」 | ✅ 游標置於中間 |
| 『』 | 『』 | ✅ 游標置於中間 |
| （）| （）| ✅ 游標置於中間 |

> 配對模式（`data-pair="true"`）：插入成對標點後，游標自動定位在兩個標點符號之間。

### 4.4 面板底部

```
┌──────────────────────────────────────────────────────┐
│  [📝 存為草稿]                        [取消] [💾 儲存公告] │
└──────────────────────────────────────────────────────┘
```

| 按鈕 | 行為 |
|------|------|
| 存為草稿 | 強制將狀態設為 `draft` 後儲存，適用於尚未準備好發佈的公告 |
| 取消 | 關閉面板，不儲存 |
| 儲存公告 | 依表單中選擇的狀態儲存（可為已發佈或已下架） |

**儲存驗證流程**：
1. 檢查公告標題（不得為空）
2. 檢查上架日期（不得為空）
3. 檢查公告內容（Quill 內容不得為空或僅含 `<p><br></p>`）
4. 通過驗證 → 儲存資料 → 關閉面板 → 刷新列表與統計卡片
5. Toast 訊息：「公告已新增」/「公告已更新」/「已存為草稿」

## 7. 自動下架機制

系統在頁面載入時（`DOMContentLoaded`）自動執行下架檢查：

```
foreach (announcement in announcementsDb)
    if status == 'published'
       AND unpublishDate != ''
       AND unpublishDate <= today
    then
        status = 'archived'
```

> **正式版建議**：改由後端排程（Background Service / Hangfire）每日自動檢查並更新狀態，避免依賴前端載入時機。

## 8. 與首頁「今日提醒」的連動

公告資料連動至首頁（firstpage.html）右側「今日提醒」看板：

| 條件 | 說明 |

| 點擊行為 | 跳轉至公告頁面或開啟公告檢視 Modal |

## 9. 操作情境 (User Scenarios)

### 情境 1：管理員發佈系統維護公告（全站廣播）

1. 管理員進入系統公告頁面
2. 點擊右上角「＋ 新增公告」
3. Slide-over 面板開啟，填寫：
   - 分類：系統公告
   - 狀態：已發佈
   - 綁定顯示梯次：全站廣播 (所有梯次)
   - 上架日期：2026-03-19
   - 下架日期：2026-03-22（維護完成後自動下架）
   - 置頂公告：✅ 勾選
4. 公告內容：
   - 標題：系統維護公告
   - 內容：使用 Quill 編輯器編寫維護時間與注意事項
5. 點擊「儲存公告」
6. 列表新增一筆置頂公告，統計卡片更新
7. 所有梯次的使用者在首頁「今日提醒」看板可看到此公告
8. 2026-03-22 後系統自動將狀態切換為「已下架」

### 情境 2：管理員編寫草稿稍後發佈

1. 管理員點擊「＋ 新增公告」
2. 填寫公告設定與內容
3. 尚未確認內容 → 點擊左下角「存為草稿」
4. 公告以「草稿」狀態儲存，列表中灰色圓點標示
5. 稍後再從列表點擊 ✏️ 編輯按鈕
6. 修改完成後將狀態改為「已發佈」→ 點擊「儲存公告」
7. 公告正式上架，首頁看板即時顯示

### 情境 3：命題教師（唯讀權限）瀏覽公告

1. 命題教師進入系統公告頁面
2. 右上角僅顯示「使用說明手冊」按鈕（無新增公告按鈕）
3. 看到公告列表，操作欄不顯示編輯 / 置頂 / 刪除按鈕
4. 點擊列表行 → 開啟檢視 Modal
5. Modal 底部無「編輯此公告」按鈕，僅有「確認關閉」
6. 點擊「使用說明手冊」→ 開啟手冊下載清單

## 13. Blazor .NET 10 遷移備註

| 項目 | 前端 DEMO 做法 | Blazor 建議做法 |
|------|---------------|----------------|
| 公告資料來源 | `announcementsDb` 陣列 | `Announcements` 資料表，透過 EF Core 讀取 |
| 梯次選項來源 | `projectsRef` 陣列 | 從 `Projects` 表動態查詢注入 `<select>` |
| Quill 編輯器 | 直接引用 CDN | Blazor JS Interop 封裝 Quill.js，或改用 Blazor 原生的 Rich Text Component（如 MudBlazor RichTextEditor） |
| 中文標點插入 | DOM 事件 + Quill API | 透過 JS Interop 呼叫 Quill `insertText`，或封裝為 Blazor Component |
| 自動下架 | 前端頁面載入時檢查 | 後端 Background Service / Hangfire 每日排程自動更新 `Status` |
| 檔案上傳 | 未實作（DEMO） | ASP.NET Core `IFormFile` + Azure Blob Storage 或本地 File Storage |
| 權限控制 | 前端按鈕隱藏 | ASP.NET Core `[Authorize(Policy = "AnnouncementEdit")]` 中介層 + 前端條件渲染 |
| 全站廣播判斷 | `project = 'ALL'` 字串 | `ProjectId IS NULL` 表示全站廣播 |
| 數字動畫 | `setInterval` 遞增 | Blazor `@inject IJSRuntime` 呼叫 JS 動畫，或純 C# Timer + `StateHasChanged()` |
