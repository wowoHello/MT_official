## 1. 頁面概述

命題任務頁面為命題教師的核心工作區，涵蓋試題的新增、編輯、預覽、送審、修題回覆與歷程檢視等完整命題生命週期。頁面以三個 TAB 分頁管理不同階段的試題，並支援 7 種題型的獨立表單結構與共用的底部滑入式 Quill 富文本編輯器。

**可操作此頁面的角色：**
- 命題教師（TEACHER）— 主要使用者
- 系統管理員（ADMIN）— 可檢視所有教師的命題

## 4. 功能需求

### 4.1 麵包屑與標題列

- 麵包屑路徑：首頁 > 命題任務
- 右側顯示當前階段指示（如「目前階段：命題階段」）
- 右上方 **新增試題** 按鈕（Sage 綠底白字 + 加號圖示）

### 4.2 我的命題配額進度

橫向排列 7 張迷你卡片，顯示教師在當前梯次被分配的各題型配額與已完成數量：

| # | 題型 | 顯示格式 |
|---|------|---------|
| 1 | 一般單選題 | `已完成 / 分配數`（如 3/150） |
| 2 | 精選單選題 | `已完成 / 分配數` |
| 3 | 閱讀題組 | `已完成 / 分配數` |
| 4 | 長文題目 | `已完成 / 分配數` |
| 5 | 短文題組 | `已完成 / 分配數` |
| 6 | 聽力測驗 | `已完成 / 分配數` |
| 7 | 聽力題組 | `已完成 / 分配數` |

- 進度條以百分比色條呈現（Morandi 藍漸層）
- 無配額時顯示灰色「0/0」

### 4.3 三個 TAB 分頁標籤

| TAB | 名稱 | 說明 |
|-----|------|------|
| compose | 命題作業區 | 管理命題階段的試題（草稿、完成、送審） |
| revision | 審修作業區 | 管理各階段退回的修題任務（審題鎖定、修題中） |
| history | 審核結果與歷史 | 查看最終判決結果（採用、不採用） |

---

### 4.4 TAB 一：命題作業區

#### 4.4.1 統計卡片（4 張，可點擊篩選）

| 卡片 | 計算邏輯 | 點擊效果 |
|------|---------|---------|
| 命題總計 | 草稿 + 完成 + 送審總和 | 清除篩選，顯示全部 |
| 命題草稿 | `status === 'draft'` | 篩選僅顯示草稿 |
| 命題完成 | `status === 'completed'` | 篩選僅顯示完成 |
| 命題送審 | `status === 'pending'` | 篩選僅顯示已送審 |

#### 4.4.2 三個狀態與可用操作

| 狀態 | 標籤顯示 | 可用操作 |
|------|---------|---------|
| **草稿** (`draft`) | 灰底灰字 | ① 編輯 ② 預覽 ③ 存為草稿 ④ 命題完成 ⑤ 刪除 |
| **命題完成** (`completed`) | 藍底藍字 | ① 編輯 ② 預覽 ③ 存為草稿（退回草稿） ④ 命題送審 |
| **已送審** (`pending`) | 黃底黃字 | ① 檢視（唯讀）② 預覽 |

> **重要**：完成後不可刪除。送審後題目鎖定為唯讀，審修前不可更改。

#### 4.4.3 列表排序優先順序

草稿優先 → 完成 → 送審 → 修改時間（新→舊）

### 4.5 TAB 二：審修作業區

#### 4.5.1 統計卡片（3 張，可點擊篩選）

| 卡片 | 計算邏輯 | 點擊效果 |
|------|---------|---------|
| 審題總計 | 鎖定 + 修題中總和 | 清除篩選，顯示全部 |
| 審題鎖定 | `peer_reviewing` / `expert_reviewing` / `final_reviewing` | 篩選僅顯示鎖定中 |
| 修題中 | `peer_editing` / `expert_editing` / `final_editing` | 篩選僅顯示修題中 |

#### 4.5.2 兩個狀態與可用操作

| 狀態 | 標籤顯示 | 可用操作 |
|------|---------|---------|
| **審題鎖定** | 藍底藍字 | 檢視（唯讀），等待審題結果 |
| **修題中** | 琥珀/紅底 | 編輯（修題模式），必須填寫「修題回覆說明」後送出 |

#### 4.5.3 修題回覆機制

命題教師在各審修階段收到退回通知後：

1. 開啟修題 Modal，頂部顯示**修題提醒橫幅**（顯示審題階段與審查意見）
2. 修改題目內容
3. **必填欄位「修題回覆說明（請說明本次修改內容）」**
4. 點擊「送出修題」

修題回覆流轉邏輯：
- 互審修題完成 → 進入專審階段
- 專審修題完成 → 進入總審階段（注意：專審退回修完直接進總審）
- 總審修題完成 → 回到總審（最多退回 2 次，第 3 次由總召自行處理）

### 4.6 TAB 三：審核結果與歷史

#### 4.6.1 統計卡片（3 張，可點擊篩選）

| 卡片 | 計算邏輯 | 點擊效果 |
|------|---------|---------|
| 全部題目 | 採用 + 不採用總和 | 清除篩選，顯示全部 |
| 已採用 | `status === 'adopted'` | 篩選僅顯示採用 |
| 不採用 | `status === 'rejected'` | 篩選僅顯示不採用 |

#### 4.6.2 兩個狀態與可用操作

| 狀態 | 標籤顯示 | 可用操作 |
|------|---------|---------|
| **採用** (`adopted`) | 翠綠底 | 檢視詳情（唯讀） |
| **不採用** (`rejected`) | 灰底 | 檢視詳情（唯讀） |

#### 4.6.3 歷程軌跡

每一題目從建立到結案的完整 LOG：

| 觸發動作 | 紀錄條件 |
|---------|---------|
| 存為草稿 | 每次內容有變動時紀錄（內容未變動不紀錄） |
| 命題完成 | 紀錄 |
| 命題送審 | 紀錄 |
| 審修調整（修題回覆送出） | 紀錄（含修題回覆說明） |
| 互審/專審/總審意見 | 紀錄（含審查意見） |
| 決策（採用/改後再審/不採用） | 紀錄 |
| 刪除（草稿階段） | 軟刪除 + 紀錄 LOG |

## 5. 新增/編輯試題 Modal（全螢幕表單）

### 5.1 Modal 結構

```
┌──────────────────────────────────────────────────────────┐
│ 題碼 Q-2603-NEW    狀態：新增中    [預覽] [存草稿] [X]    │
├─────────────┬────────────────────────────────────────────┤
│ 左側屬性欄   │ 右側主要編輯區                              │
│ (可收合)     │                                            │
│              │ ┌────────────────────────────────────────┐ │
│ ▸ 題型       │ │ 題幹                                    │ │
│   一般單選題  │ │ [點擊此處開啟編輯器]                     │ │
│              │ ├────────────────────────────────────────┤ │
│ ▸ 等級       │ │ 選項                                    │ │
│   中級       │ │ ○ A: ___________                       │ │
│              │ │ ● B: ___________  ← 正確答案            │ │
│ ▸ 分類       │ │ ○ C: ___________                       │ │
│   語詞       │ │ ○ D: ___________                       │ │
│              │ ├────────────────────────────────────────┤ │
│ ▸ 難度       │ │ 解析                                    │ │
│   中         │ │ [點擊此處開啟編輯器]                     │ │
│              │ └────────────────────────────────────────┘ │
│ [收合 ◂]     │                                            │
├─────────────┴────────────────────────────────────────────┤
│ ████████████ 底部滑入式 Quill 編輯器 ████████████████████  │
│ [工具列] [中文標點符號快速插入列]                           │
│ [富文本編輯區]                                             │
└──────────────────────────────────────────────────────────┘
```

### 5.2 表單模式

| 模式 | 觸發方式 | 行為 |
|------|---------|------|
| `create` | 點擊「新增試題」 | 空白表單，所有欄位可編輯 |
| `edit` | 點擊草稿/完成試題的「編輯」 | 帶入既有資料，所有欄位可編輯 |
| `revision` | 點擊修題中試題的「編輯」 | 帶入資料 + 頂部修題提醒橫幅 + 必填修題回覆 |
| `view` | 點擊已送審/鎖定/歷史的「檢視」 | 所有輸入框唯讀灰色 |

### 5.3 左側屬性欄（可收合）

點擊收合按鈕可將左側欄收起，專注於右側編輯區。

#### 5.3.1 通用屬性

| 欄位 | 類型 | 說明 |
|------|------|------|
| 題型 | Select | 7 種題型擇一（變更題型會重置右側表單） |
| 等級 | Select | 依題型動態切換選項（一般等級 / 聽力難度） |
| 難度 | Select | 易/中/難 |

#### 5.3.2 題型專屬屬性

依選擇的題型，左側動態顯示不同的專屬屬性欄位：

**一般單選題 / 精選單選題：**

| 欄位 | 說明 |
|------|------|
| 分類（主類） | 文字/語詞/成語短語/造句標點/修辭技巧/語文知識/文意判讀 |
| 分類（細目） | 依主類連動的子選項 |

**長文題目：**

| 欄位 | 說明 |
|------|------|
| 作文模式 | 引導寫作 / 資訊整合 |

**短文題組：**

| 欄位 | 說明 |
|------|------|
| 主要分類 | 文義判讀（固定） |
| 次要分類 | 篇章辨析（固定） |
| 文體 | 文言文 / 應用文 / 語體文 |

**聽力測驗：**

| 欄位 | 說明 |
|------|------|
| 核心能力 | 依等級自動連動（如難度一→提取訊息） |
| 細目指標 | 依等級自動連動 |
| 音訊類型 | 對話 / 情境 / 陳述 |
| 素材來源 | 生活 / 教育 / 職場 / 專業 |

**聽力題組：**
- 子題各自帶有固定的等級/核心能力/指標設定

**閱讀題組：**
- 無額外專屬屬性

### 5.4 右側主要編輯區

依題型不同，右側編輯區動態渲染對應的欄位組合：

#### 5.4.1 題型欄位契約

| 題型 | 題幹 | 選項 ABCD | 文章/段落 | 子題 | 音檔 | 解析 |
|------|------|----------|----------|------|------|------|
| 一般單選題 | ✓ | ✓ | ✗ | ✗ | ✗ | ✓ |
| 精選單選題 | ✓ | ✓ | ✗ | ✗ | ✗ | ✓ |
| 長文題目 | ✓（題目） | ✗ | ✓（文章內容） | ✗ | ✗ | ✓（批閱說明） |
| 閱讀題組 | ✓（標題） | ✗ | ✓（文章內容） | ✓（選擇題） | ✗ | ✓（題組解析） |
| 短文題組 | ✓（題目） | ✗ | ✓（文章內容） | ✓（論述題） | ✗ | ✓（題組解析） |
| 聽力測驗 | ✓ | ✓ | ✗ | ✗ | ✓ | ✓ |
| 聽力題組 | ✗ | ✗ | ✓（語音內容） | ✓（選擇題，固定2題） | ✓ | ✓（題組解析） |

#### 5.4.2 選項區塊（ABCD 選擇題）

- 4 個選項（A/B/C/D），每個選項前有圓形 Radio 按鈕
- 點擊 Radio 設定正確答案（點亮為綠色填滿）
- 選項文字點擊後開啟底部 Quill 編輯器
- 正確答案綠色背景高亮

#### 5.4.3 子題管理（題組類型）

**選擇題模式（閱讀題組、聽力題組）：**
- 每個子題包含：題幹 + ABCD 選項 + 正確答案
- 可新增/移除子題（聽力題組固定 2 題，不可增減）
- 正確答案下拉與子題內容雙向同步

**論述題模式（短文題組）：**
- 每個子題包含：題幹 + 解析
- 子題各自帶有「向度」與「細目指標」屬性下拉

### 5.5 底部滑入式 Quill 編輯器

- 點擊任一文字輸入框時，從底部滑入展開
- 工具列包含：粗體/斜體/底線/刪除線、標題層級、列表、縮排、對齊、連結、圖片、清除格式
- **中文標點符號快速插入列**：，。、；：？！「」『』（）——……《》〈〉
- 編輯器內容與對應的輸入框即時雙向同步
- 點擊 Modal 其他區域時，編輯器收回

### 5.6 預覽 Modal

- 模擬考試卷樣式的題目預覽
- 顯示題目在考卷上的呈現模樣
- 包含題號、題幹、選項排版
- 唯讀不可編輯

### 5.7 操作按鈕邏輯

#### 5.7.1 新增/編輯模式

| 按鈕 | 行為 |
|------|------|
| 存為草稿 | 儲存當前內容為草稿狀態，留在 Modal 或關閉 |
| 命題完成 | 驗證必填欄位 → 狀態改為 `completed` → 關閉 Modal |
| 命題送審 | 僅 `completed` 狀態可用 → 狀態改為 `pending` → 鎖定 |
| 預覽 | 開啟考卷預覽 Modal |
| 刪除 | 僅 `draft` 狀態可用 → 二次確認 → 軟刪除 + 紀錄 LOG |

#### 5.7.2 修題模式

| 按鈕 | 行為 |
|------|------|
| 送出修題 | 驗證必填欄位 + 修題回覆說明不可為空 → 送出 |
| 預覽 | 開啟考卷預覽 |

#### 5.7.3 點擊 Modal 外部背景

- **新增/編輯模式**：自動存為草稿並關閉
- **檢視模式**：僅關閉不儲存

| 向度 | 細目指標 |
|------|---------|
| 條列敘述 | 1-1 條列敘述人、事、物特徵與特質<br>1-2 條列敘述起始原因、發生情況、結論等時空先後順序<br>1-3 條列敘述人、事、物的差異 |
| 歸納統整 | 2-1 歸納作者主張<br>2-2 歸納文章主旨<br>2-3 歸納共同特點 |
| 分析推理 | 3-1 分析線索<br>3-2 推論緣由<br>3-3 判斷結果<br>3-4 判斷詞性、主語<br>3-5 判斷字句的解釋、文意說明是否正確<br>3-6 推測行為的原因或用意<br>3-7 推測寫作手法的目的<br>3-8 判斷文體、格律、風格 |

## 7. 使用情境

### 情境：命題教師進行命題工作

> 身分：命題教師，被指派至 115年度春季全民中檢，目前為「命題階段」

1. **新增試題**：點擊【新增試題】→ 全螢幕 Modal 開啟
2. **設定屬性**：在左側選擇題型（如一般單選題）、等級（中級）、分類（語詞/詞彙辨析）、難度（中）
3. **收合左側**：點擊收合標籤將屬性欄收起，專注於編輯
4. **編輯題目**：
   - 輸入題幹（點擊文字框，底部 Quill 編輯器滑入）
   - 輸入 A/B/C/D 四個選項
   - 點亮正確答案前方的圓圈（如 B 選項）
   - 撰寫解析
5. **預覽**：點擊【預覽】查看考卷樣式呈現
6. **存為草稿**：尚未確定，先點擊【存為草稿】→ 歷程軌跡開始紀錄 → 題目出現在命題作業區列表
7. **確定完成**：再次編輯後點擊【命題完成】→ 狀態改為「命題完成」
8. **送審**：確認無誤後點擊【命題送審】→ 狀態改為「已送審」→ 題目鎖定為唯讀

### 後續審修情境

9. 收到互審退回通知 → 題目出現在「審修作業區」→ 狀態為「互審修題」
10. 點擊編輯進入修題模式 → 查看審查意見 → 修改內容 → **填寫修題回覆說明** → 送出
11. 專審退回 → 再次修題回覆送出 → 進入總審
12. 最終結果出現在「審核結果與歷史」→ 檢視完整歷程軌跡

## 9. API 端點規劃（Blazor .NET 10 遷移參考）

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/cwt/quotas/{projectId}/{userId}` | 取得教師在指定梯次的配額進度 |

## 10. 已知限制與改進方向

| 項目 | 現況（Demo） | 遷移後目標 |
|------|-------------|-----------|
| 試題資料 | 前端寫死假資料 | 後端 CRUD API + 資料庫 |
| 配額資料 | 前端寫死 teacherQuotaDb | 從 ProjectMembers + MemberQuotas 查詢 |
| Quill 編輯器 | 前端直接載入 Quill.js | 評估 Blazored.TextEditor 或保留 JS Interop |
| 自動存草稿 | 點擊背景關閉時存草稿 | 可加入定時自動存草稿（每 30 秒） |
| 分頁 | 前端分頁（defaultPageSize=12） | 後端分頁 |
| 圖片上傳 | Quill Base64 內嵌 | 後端圖片上傳 API + CDN |
| 音檔上傳 | 僅顯示 Demo 佔位 | 實際音檔上傳功能 |
| 修題回覆 | 前端模擬 | 後端儲存 + 通知審查人員 |
| 軟刪除 | 前端陣列移除 | 後端 IsDeleted 標記 + 不顯示在列表 |

**Blazor 遷移特別注意：**
> Quill.js 為前端核心依賴。遷移時建議：
> 1. 使用 Blazored.TextEditor 套件（Quill 的 Blazor 封裝）
> 2. 或透過 JSInterop 保留原有 Quill 實例
> 中文標點符號快速插入列需自行擴充 Quill toolbar
