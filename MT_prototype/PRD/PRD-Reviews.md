## 1. 頁面概述

審題任務頁面為三審制度（互審 → 專審 → 總審）的核心工作區，提供審題委員與相關人員對命題試題進行審閱、給予意見或做出決策。頁面以雙 TAB 分頁管理進行中的審題工作與已結案的審核歷史。

**可操作此頁面的角色：**
- 命題教師（TEACHER）— 互審階段給予意見
- 審題委員/專家學者（REVIEWER）— 專審階段審閱
- 總召（CHIEF）— 總審階段最終裁決

## 4. 功能需求

### 4.1 麵包屑與標題列

- 麵包屑路徑：首頁 > 審題任務
- 右側顯示當前階段 Badge（如「交互審題進行中」）
- 當前階段 Badge 根據 `getCurrentStageInfo()` 動態計算

### 4.2 雙 TAB 分頁標籤

| TAB | 名稱 | 說明 |
|-----|------|------|
| review | 審題作業區 | 進行中的審題任務（互審/專審/總審，含已決策） |
| history | 審核結果與歷史 | 最終判決結果（採用/不採用） |

### 4.3 TAB 一：審題作業區

#### 4.3.1 七階段時程圖

水平排列 7 個階段節點，以圓點 + 連線呈現：

| # | 階段 Key | 顯示名稱 |
|---|---------|---------|
| 1 | proposing | 命題 |
| 2 | peerReview | 互審 |
| 3 | peerEdit | 互修 |
| 4 | expertReview | 專審 |
| 5 | expertEdit | 專修 |
| 6 | finalReview | 總審 |
| 7 | finalEdit | 總修 |

**圓點狀態規則：**

| 狀態 | 條件 | 視覺效果 |
|------|------|---------|
| 已完成 | `EndDate < 今天` | 藍色實心圓（Morandi 藍） |
| 進行中 | `StartDate ≤ 今天 ≤ EndDate` | 綠色圓 + **呼吸燈動畫**（`stage-pulse` 2s 無限循環） |
| 未開始 | `StartDate > 今天` | 灰色空心圓 |

- 時程底下顯示小文字：「目前階段：交互審題」
- 顯示距離當前階段結束的剩餘天數

#### 4.3.2 統計計數（3 項）

| 計數項 | 計算邏輯 |
|-------|---------|
| 本區題目 | 審題作業區所有題目總數 |
| 待處理 | `reviewStatus === 'pending'` 的題目數 |
| 已決策 | `reviewStatus === 'decided'` 的題目數 |

#### 4.3.3 篩選功能

| 篩選器 | 類型 | 選項 |
|-------|------|------|
| 題號/關鍵字 | 文字輸入（模糊搜尋） | 比對題號、題幹、文章、選項、解析 |
| 題型 | 下拉選單 | 全部 / 7 種題型 |
| 等級 | 下拉選單 | 全部 / 初級～優級 / 難度一～五 |
| 狀態 | 下拉選單 | 所有狀態 / 互審待審 / 專審待審 / 總審待審 / 已決策 |

右上角顯示分頁器與每頁筆數選擇（12/24/48），旁邊顯示「共 X 題」。

#### 4.3.4 列表項目（全寬卡片式，整列可點擊展開）

每張卡片由上而下排列以下資訊：

**第一行（Meta 列）：**

| 元素 | 位置 | 說明 |
|------|------|------|
| 題型圖示 | 最左側 | FontAwesome icon，依題型不同（圓點/星/書本/文件/層疊/耳機） |
| 題號 | 圖示右方 | 如 `Q-2602-R017`，粗體 |
| 題型名稱 | 題號後 | 如「閱讀題組」，灰色小字 |
| 等級/難度 | 題型後 | 如「中級 / 中」，灰色小字 |
| 審查階段 Badge | 名稱後 | 含圖示：互審 👥（藍底藍字）/ 專審 👤（紫底紫字）/ 總審 👑（紅底紅字） |

**第二行（內容預覽列）：**
- 顯示題幹或文章內容的前 1~2 行截斷文字預覽
- 使用 `line-clamp` 截斷，超出顯示 `...`
- 題組類型顯示文章段落預覽而非子題題幹

**第三行（底部資訊列）：**

| 元素 | 位置 | 說明 |
|------|------|------|
| 命題教師 | 左下 | 「命題教師：陳美玲」，灰色小字 |
| 操作按鈕 | 右側 | **審題**（綠底白字）/ **檢視**（灰底）→ 按鈕垂直置中於整張卡片 |

**操作按鈕規則：**
- **審題**：未決策 + 當前處於對應審查階段 → 綠色按鈕 `✏ 審題`
- **檢視**：已決策，或尚未輪到對應階段 → 灰色按鈕

**排序優先順序：**
- 階段優先：總審 > 專審 > 互審
- 狀態優先：待處理 > 已決策

## 5. 三審制度：決策邏輯與按鈕規則

### 5.1 各階段可用決策

| 審查階段 | 審題角色 | 可用按鈕 | 說明 |
|---------|---------|---------|------|
| **互審** | 命題教師 | **送出意見** | 僅給予意見，**無**採用/改後再審/不採用按鈕 |
| **專審** | 專家學者 | **採用**、**改後再審** | **無**不採用按鈕 |
| **總審** | 總召 | **採用**、**改後再審**、**不採用** | 最後決策權 |

### 5.2 決策後流轉

| 決策 | 階段 | 後續流向 |
|------|------|---------|
| 送出意見（互審） | peer | 題目進入互審修題 → 命題教師修改 → 進入專審 |
| 採用（專審） | expert | 題目直接進入總審階段 |
| 改後再審（專審） | expert | 退回命題教師修改 → 修改後**直接進入總審** |
| 採用（總審） | final | 直接進入「審核結果與歷史」→ 採用，後續不再開放編輯 |
| 改後再審（總審） | final | 退回命題教師修改（進入總審修題階段），來回修改第三次則由總召自行修改，不再有此按鈕 |
| 不採用（總審） | final | 退回命題教師修改（進入總審修題階段），來回修改第三次則由總召自行修改，若此次點按不採用則直接進入「審核結果與歷史」→ 不採用，後續不再開放編輯 |

### 5.3 總審退回次數限制

| 退回次數 | 行為 |
|---------|------|
| 第 1 次退回 | 命題教師修改後再次送審至總審 |
| 第 2 次退回 | 命題教師修改後再次送審至總審 |
| 第 3 次（不再退回） | 總審取得**編輯權限**，可自行修改題目內容，直接判定「採用」或「不採用」 |

### 5.4 迴避控制邏輯

| 規則 | 說明 |
|------|------|
| 自己不審自己 | 命題教師不會被分配到自己命的題目進行互審/專審 |
| 總審隔離專審 | 若總召同時擔任專審委員，進入總審時不會取得自己在專審已審過的題目，且該梯次一定會有兩位以上總召 |

### 5.5 操作按鈕解鎖規則

| 審題者身分 | 當前階段為其對應階段 | 按鈕狀態 |
|-----------|-------------------|---------|
| 互審教師 | ✓ 在互審階段 | **審題**（可操作） |
| 互審教師 | ✗ 非互審階段 | **檢視**（唯讀） |
| 專審委員 | ✓ 在專審階段 | **審題**（可操作） |
| 專審委員 | ✗ 非專審階段 | **檢視**（唯讀） |
| 總召 | ✓ 在總審階段 | **審題**（可操作） |
| 任何人 | 已決策 | **檢視**（唯讀，不可再次更改） |

## 6. 審查意見輸入區

### 6.1 手風琴展開區

- 預設收合狀態
- 標題列：「審查意見」文字 + 右側「試題比對」按鈕 + 展開/收合箭頭

### 6.2 罐頭訊息（快速插入）

7 則預設審查意見，以按鈕列呈現，點擊即插入 Quill 編輯器游標處：

| # | 罐頭訊息內容 |
|---|-------------|
| 1 | 題目設計清晰，符合等級標準。 |
| 2 | 建議調整選項鑑別度，增加迷惑選項難度。 |
| 3 | 解析內容過於簡略，建議補充說明。 |
| 4 | 題幹語意模糊，建議修正敘述方式。 |
| 5 | 正確答案可能有爭議，請確認並補充理據。 |
| 6 | 選項排列建議依照邏輯或筆畫順序。 |
| 7 | 文章篇幅適中，與子題配合良好。 |

### 6.3 中文標點符號快速插入列

與命題任務共用的標點符號工具：
- 單字符：，。、；：！？——……
- 成對符號（插入一對，游標置中）：「」『』（）

### 6.4 Quill 編輯器

- Snow 主題
- 工具列：粗體/斜體/底線、標題、列表、縮排、對齊、連結、清除格式
- 自訂字型：標準、標楷體、Times New Roman
- 高度約 140px

### 6.5 試題比對（重複度檢查）

- 點擊「試題比對」按鈕開啟 SweetAlert2 Modal
- 顯示比對結果表格：

| Q-2501-045 | 42% | 低度相似 |
| Q-2502-112 | 18% | 無疑慮 |
| Q-2503-078 | 12% | 無疑慮 |

- 目前為 Demo 模擬結果，實際由後端 AI 引擎執行

## 8. 使用情境

### 情境一：命題教師進行互審

> 身分：命題教師，命題階段結束，進入互審階段

1. 進入審題任務頁面 → 查看七階段時程圖，呼吸燈在「互審」位置
2. 在審題作業區看到被分配的待審題目
3. 點擊某題的【審題】按鈕展開詳情
4. 仔細閱讀題目、選項、解析
5. 展開「審查意見」手風琴
6. 使用罐頭訊息或自行輸入審查意見
7. 點擊【送出意見】
8. 該題進入「已決策」狀態，僅能檢視，不可再更改

### 情境二：審題委員等待專審階段

> 身分：審題委員（專審），目前在互審修題階段

1. 進入審題任務頁面 → 查看呼吸燈在「互修」位置
2. 審題作業區顯示被分配的所有題目
3. 因為不是專審階段，所有操作按鈕僅為【檢視】
4. 等待階段進入「專家審題」後，【審題】按鈕才會解鎖
5. 解鎖後可進行審題，決策按鈕為：【採用】或【改後再審】

### 情境三：總召進行總審與退回機制

> 身分：總召人員，進入總召審題階段

1. 進入審題任務頁面 → 查看呼吸燈在「總審」位置
2. 點擊【審題】進入審題
3. 查看題目內容、答案、解析
4. 查看底部「歷程軌跡與審查意見」，了解互審/專審的意見
5. 輸入總召審查意見
6. 做出決策：

**路徑 A — 採用：**
- 點擊【採用】→ 題目直接進入「審核結果與歷史」TAB
- 試題鎖定，後續不再開放編輯

**路徑 B — 改後再審（第 1~2 次）：**
- 點擊【改後再審】→ 題目退回命題教師
- 進入「總審修題」階段 → 命題教師修改後再送審
- 總召再次審閱 → 可再次「採用」或「改後再審」
- 第 2 次退回後，命題教師仍可修改再送審

**路徑 C — 第 3 次不通過：**
- 已退回 2 次 → 第 3 次總召取得**編輯權限**
- 總召可自行修改題目內容
- 直接判定「採用」（以總召修改後版本）或「不採用」
- **不再退回命題教師**

**路徑 D — 不採用：**
- 點擊【不採用】→ ※ 與改後再審流程相同，不採用狀態只是告知命題老師這題嚴重性

## 9. 資料庫資料表規劃

### 9.1 ReviewAssignments（審題分配表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| QuestionId | UNIQUEIDENTIFIER | FK → Questions.Id, NOT NULL | 被審題目 |
| ProjectId | UNIQUEIDENTIFIER | FK → Projects.Id, NOT NULL | 所屬梯次 |
| ReviewerId | UNIQUEIDENTIFIER | FK → Users.Id, NOT NULL | 審題者 |
| ReviewStage | NVARCHAR(10) | NOT NULL | 審查階段（peer/expert/final） |
| ReviewStatus | NVARCHAR(10) | NOT NULL, DEFAULT 'pending' | 審查狀態（pending/decided） |
| Decision | NVARCHAR(10) | NULL | 決策結果（comment/adopt/revise/reject） |
| Comment | NVARCHAR(MAX) | NULL | 審查意見內容（HTML） |
| DecidedAt | DATETIME2 | NULL | 決策時間 |
| CreatedAt | DATETIME2 | DEFAULT GETUTCDATE() | 分配時間 |

**UNIQUE 約束：** `(QuestionId, ReviewerId, ReviewStage)` 聯合唯一

### 9.2 ReviewReturnCounts（總審退回次數追蹤表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| QuestionId | UNIQUEIDENTIFIER | FK → Questions.Id, NOT NULL | 被退回題目 |
| FinalReviewerId | UNIQUEIDENTIFIER | FK → Users.Id, NOT NULL | 總審審題者 |
| ReturnCount | INT | NOT NULL, DEFAULT 0 | 退回次數（最大 2，第 3 次不退回） |
| CanEditByReviewer | BIT | DEFAULT 0 | 總審是否取得編輯權限（ReturnCount ≥ 2 時為 1） |

### 9.3 SimilarityChecks（試題比對記錄表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| SourceQuestionId | UNIQUEIDENTIFIER | FK → Questions.Id, NOT NULL | 來源題目 |
| ComparedQuestionId | UNIQUEIDENTIFIER | FK → Questions.Id, NOT NULL | 比對題目 |
| SimilarityScore | DECIMAL(5,2) | NOT NULL | 相似度百分比 |
| Determination | NVARCHAR(20) | NOT NULL | 判定結果（無疑慮/低度相似/高度相似） |
| CheckedAt | DATETIME2 | DEFAULT GETUTCDATE() | 比對時間 |
| CheckedBy | UNIQUEIDENTIFIER | FK → Users.Id | 發起比對者 |

### 9.4 CannedMessages（罐頭訊息表，可管理化）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| Content | NVARCHAR(500) | NOT NULL | 訊息內容 |
| SortOrder | INT | NOT NULL, DEFAULT 0 | 排序順序 |
| IsActive | BIT | DEFAULT 1 | 是否啟用 |

## 10. API 端點規劃（Blazor .NET 10 遷移參考）

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/reviews?projectId=&tab=&filters=` | 取得審題列表（含分頁、篩選） |
| GET | `/api/reviews/{assignmentId}` | 取得單一審題詳情（含題目、歷程） |
| POST | `/api/reviews/{assignmentId}/decision` | 送出審查決策（comment/adopt/revise/reject） |
| GET | `/api/reviews/stage-info/{projectId}` | 取得當前梯次的階段時程資訊 |
| GET | `/api/reviews/{questionId}/history` | 取得題目的完整歷程軌跡 |
| POST | `/api/reviews/{questionId}/similarity-check` | 執行試題比對（呼叫 AI 引擎） |
| GET | `/api/reviews/canned-messages` | 取得罐頭訊息列表 |
| PUT | `/api/reviews/{assignmentId}/edit-question` | 總審第 3 次直接編輯題目 |
