> **文件版本**：v1.0

---

## 1. 頁面概述

角色與權限管理頁面為 CWT 命題工作平臺的系統管理核心，僅限系統管理員（ADMIN）存取。頁面提供兩大 TAB 頁籤：

- **人員帳號管理**：維護內部人員（CWT 公司職員）帳號的新增、編輯、停用/啟用、密碼重設
- **角色權限管理**：定義並維護系統中所有角色（含內部人員與外部人員），設定各角色可存取的功能區塊

> **關鍵概念**：此頁面僅管理「內部人員」帳號。外部人員（命題教師、審題委員）的帳號由「教師管理系統」頁面維護，並透過角色權限管理中定義的外部人員角色來控制其存取範圍。

## 2. 使用者角色與存取權限

| 角色 | 可存取此頁面 | 可執行操作 |

### 4.3 新增 / 編輯人員（Slide-over 面板）

點擊「＋ 新增人員」或詳情面板「編輯資料」按鈕，從右側滑入 Slide-over 面板。

**面板結構分為 2 個區塊**：

#### 區塊 1：基本資料

| 欄位 | 類型 | 必填 | 驗證規則 | 說明 |
|------|------|:----:|---------|------|
| 使用者姓名 | text | ✅ | 不得為空 | placeholder「如：王小明」 |
| 登入帳號 | text | ✅ | 不得為空、不可重複 | placeholder「如：admin01」；**編輯模式下禁用（不可修改）** |
| 電子信箱 | email | | email 格式 | placeholder「name@company.com.tw」 |
| 身份別 | select | ✅ | 必須選擇 | **僅列出「內部人員」分類的角色**，預設角色後方標註「(預設)」 |
| 公司職稱 | text | | | placeholder「如：專案經理」 |

#### 區塊 2：帳號設定

| 欄位 | 類型 | 說明 |
|------|------|------|
| 預設密碼提示 | info | 藍底提示框：「預設密碼為公司統編 `01024304`，首次登入時系統會強制要求變更密碼。」 |

| 備註 | textarea | 3 行，placeholder「如：特殊權限需求、備忘事項...」 |

**面板底部**：「取消」與「儲存人員」按鈕

**儲存驗證流程**：
1. 檢查必填欄位（姓名、帳號、身份別）
2. 新增模式：帳號名稱不可重複（不分大小寫）
3. 通過驗證 → 儲存資料 → 關閉面板 → 刷新列表與統計卡片
4. Toast 訊息：「人員『王小明』已成功新增。」（右上角 2 秒自動消失）

### 5.4 新增 / 編輯角色（Modal 對話框）

點擊「＋ 新增角色」或卡片上的「編輯 / 檢視」按鈕，以居中 Modal 方式開啟。

**Modal 結構分為 2 個區塊**：

#### 區塊 1：角色資訊

| 欄位 | 類型 | 必填 | 說明 |
|------|------|:----:|------|
| 角色名稱 | text | ✅ | placeholder「如：系統管理員」 |
| 分類 | select | ✅ | 「內部人員」/「外部人員」 |
| 角色描述 | textarea | | 2 行，placeholder「此角色的用途描述...」 |

> ⚠️ 預設角色開啟 Modal 時，上述三欄全部 `disabled`，並顯示提示「🔒 預設角色，權限不可修改」。

#### 區塊 2：功能區塊權限

以 Toggle 開關列表呈現 **8 個功能模組**：

| Key | 功能名稱 | 圖示 | 對應頁面 |
|-----|---------|------|---------|
| `dashboard` | 命題儀表板 | `fa-chart-pie` | dashboard.html |
| `projects` | 命題專案管理 | `fa-box-archive` | projects.html |
| `overview` | 命題總覽 | `fa-globe` | overview.html |
| `compose` | 命題任務 | `fa-pen-to-square` | cwt-list.html |
| `review` | 審題任務 | `fa-magnifying-glass-chart` | reviews.html |
| `teachers` | 教師管理系統 | `fa-chalkboard-user` | teachers.html |
| `roles` | 角色與權限管理 | `fa-id-card-clip` | roles.html |
| `announcements` | 系統公告/使用說明 | `fa-bullhorn` | announcements.html |

**Toggle 開關行為**：
- 滑動開關切換該功能的存取權限（ON / OFF）
- 每行包含圖示、功能名稱、對應頁面檔名（小字灰色）
- 預設角色：所有 Toggle 設為 `disabled` 且半透明

**進階權限：公告權限層級**

- 僅當「系統公告/使用說明」Toggle 為 ON 時顯示
- 縮排於 Toggle 下方
- 選項：「僅瀏覽 (View Only)」/「瀏覽與編輯 (Edit)」
- 預設值：僅瀏覽

**儲存驗證流程**：
1. 檢查必填欄位（角色名稱）
2. 收集所有 Toggle 開關的 ON/OFF 狀態
3. 若公告 Toggle 為 ON，讀取公告權限層級
4. 通過驗證 → 儲存角色 → 關閉 Modal → 刷新角色卡片、統計卡片、人員列表角色下拉
5. Toast 訊息：「角色『計畫主持人』已成功新增。」

1. 管理員進入角色與權限管理頁面，預設處於「人員帳號管理」TAB
2. 點擊右上角「＋ 新增人員」按鈕
3. Slide-over 面板從右側滑入
4. 填寫資料：
   - 姓名：王小明

   - 身份別：選擇「計畫主持人」
   - 公司職稱：專案經理

   - 命題儀表板 → ON
   - 命題總覽 → ON
   - 系統公告/使用說明 → ON，公告權限層級 → 僅瀏覽
   - 其餘 → OFF
6. 點擊「儲存角色」
7. 角色卡片列表新增一張紫色底自訂角色卡片
8. 統計卡片「角色總數」+1、「內部人員角色」+1
9. 人員帳號管理的「身份別」下拉選單自動包含此新角色

### 情境 3：管理員停用離職人員帳號並重設密碼

1. 管理員在人員帳號管理左側列表搜尋「王美玲」
2. 點擊該筆人員，右側載入詳情
3. 確認此人已無進行中任務
4. 點擊「停用帳號」→ SweetAlert2 確認對話框
5. 確認後，帳號狀態切換為「停用」
6. 列表項目圓點變紅，詳情 Badge 更新為紅色「停用」
7. 統計卡片「啟用中」-1、「停用中」+1
8. （可選）點擊「重設密碼」→ 確認後密碼恢復為預設值 `01024304`
9. 該帳號下次重新啟用後登入，將強制要求變更密碼

## 7. 資料庫規劃

### 7.1 資料表關聯圖

```
┌──────────────────┐     ┌──────────────────┐     ┌───────────────────┐

│──────────────────│     │──────────────────│     │───────────────────│
│ Id (PK)          │◄─┐  │ Id (PK)          │     │ Id (PK)           │
│ Name             │  │  │ Username         │     │ RoleId (FK→Roles) │
```

## 9. 預設密碼與首次登入機制

| 項目 | 說明 |
|------|------|
| 預設密碼 | 公司統一編號 `01024304` |
| 適用對象 | 內部人員帳號（由本頁面新增） |
| 首次登入強制變更 | 新建帳號 `IsFirstLogin = true`，登入後系統彈出強制變更密碼 Modal，變更完成後設為 `false` |
| 重設密碼 | 管理員點擊「重設密碼」後，密碼恢復為 `01024304`，`IsFirstLogin` 重設為 `true` |

> **注意**：外部人員（命題教師、審題委員）的預設密碼機制不同，詳見 PRD-Teachers.md。

## 11. 與其他頁面的交互關係

## 12. Blazor .NET 10 遷移備註

| 項目 | 前端 DEMO 做法 | Blazor 建議做法 |
|------|---------------|----------------|
| 角色資料來源 | `mockRoles` 陣列 | `Roles` + `RolePermissions` 資料表，透過 EF Core 讀取 |
| 帳號資料來源 | `mockAccounts` 陣列 | `Users` 表 JOIN `Roles` 表 |
| 預設密碼 | 寫死於前端提示 | 後端 `appsettings.json` 配置，密碼以 bcrypt/PBKDF2 雜湊儲存 |
| Toggle 開關 | DOM class toggle | Blazor `InputCheckbox` 雙向綁定 `bool` 屬性 |
| Slide-over 動畫 | CSS transition + JS | Blazor `@if` + CSS transition 或 Mudblazor Drawer 元件 |
| Modal 對話框 | DOM hidden/show + CSS | Blazor `DialogService` 或自訂 Component |
| 首頁功能卡片過濾 | 前端 `menuItems` + `roles` 陣列 | 後端 API 回傳使用者可見的 Module 清單，前端據此渲染 |
| 權限中介層 | 無（僅前端判斷） | ASP.NET Core Authorization Policy + `[Authorize(Policy = "...")]` |
