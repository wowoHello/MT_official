## 1. 頁面概述

教師管理系統（人才庫）為系統管理員管理所有命題與審題相關人員的核心模組。提供教師名單的新增、編輯、檢視、帳號啟停、密碼重設，以及跨專案的命題歷程追蹤與專案指派功能。本頁面同時也是「命題專案管理」指派人員時的人員來源。

**可操作此頁面的角色：** 系統管理員（ADMIN）

## 4. 功能需求

### 4.1 麵包屑與標題列

- 麵包屑路徑：首頁 > 教師管理系統

- 右上方按鈕：
  - **匯出名單**：匯出本梯次參與的教師名單（CSV/Excel）

| 卡片 | 計算邏輯 | 圖示顏色 |
|------|---------|---------|
| 人才庫總數 | 所有教師（含啟用+停用）總數 | Morandi 藍 |
| 帳號啟用中 | `accountStatus === 'active'` 總數 | Sage 綠 |
| 帳號停用中 | `accountStatus === 'inactive'` 總數 | Terracotta 紅橘 |
| 本梯次參與教師 | 當前梯次中有被指派的教師數 | Morandi 藍 |

- 「本梯次參與教師」會根據頂部導覽列的梯次切換動態更新

### 4.3 左側：教師列表面板

## 6. 帳號與密碼管理

### 6.1 帳號啟停

| 操作 | 按鈕樣式 | 確認方式 | 效果 |
|------|---------|---------|------|
| 停用帳號 | Terracotta 紅底 + 🔒 圖示 | SweetAlert2 確認 | `accountStatus → 'inactive'`，教師無法登入 |
| 啟用帳號 | Sage 綠底 + ✓ 圖示 | SweetAlert2 確認 | `accountStatus → 'active'`，教師恢復登入 |

### 6.2 重設密碼

- 將密碼重設為預設密碼：`Cwt2026!`
- 設定 `firstLogin = true`
- 下次登入時系統強制要求變更密碼
- SweetAlert2 成功提示：「下次登入時系統將要求變更密碼」

### 6.3 首次登入機制

- 新增教師時 `firstLogin = true`
- 教師首次以預設密碼登入後，系統強制導向至變更密碼頁面
- 變更完成後 `firstLogin = false`

## 7. 特殊情境：內部人員與人才庫帳號合併

### 7.1 問題描述

系統管理員（如總召）需要同時擁有：
- **內部人員帳號**：透過「角色與權限管理」建立，用於管理後台
- **人才庫帳號**：透過「教師管理系統」建立，用於被指派為總召/審題委員

若同一人分別在兩處建立帳號，會產生兩組帳密，造成管理困擾。

### 7.2 解決方案：帳號自動合併機制

**規則：** 在教師管理系統新增教師時，若輸入的電子信箱（帳號）已存在於「角色與權限管理」的內部人員帳號中，系統自動將兩者連結為**同一個使用者帳號**。

**合併邏輯：**

```
新增教師 → 輸入 Email
    ↓
檢查 Users 表是否已存在相同 Email
    ↓
  ┌─ 已存在（內部人員）→ 不建立新帳號，將既有帳號加入人才庫
  │    → 共用同一組帳密登入
  │    → 在人才庫中顯示此教師資料
  │    → 可被指派至專案擔任總召/專審等角色
  │
  └─ 不存在 → 正常建立新帳號（預設密碼 Cwt2026!）
```

**效果：**
- 管理員使用原本的內部帳密即可登入
- 登入後根據權限同時看到管理功能與審題任務
- 人才庫中的教師資料與內部人員帳號共享 Users 表同一筆記錄

## 8. 使用情境

### 情境一：新增教師並指派至專案

1. 管理員點擊【新增教師】→ 滑入面板開啟
2. 填寫基本資料（姓名、信箱、學校等）
3. 帳號設定保持預設「啟用」
4. 點擊【儲存教師】→ 教師出現在左側列表
5. 點擊該教師 → 右側顯示詳情
6. 切換至「參與專案」TAB → 點擊【加入梯次】
7. 選擇專案 + 勾選角色（如「命題教師」「互審教師」）
8. 確認加入 → 教師出現在該專案的人員列表

### 情境二：查看教師命題歷程

1. 管理員在左側列表點擊某位教師
2. 切換至「命題歷程」TAB
3. 查看統計小卡：累計產出 7 題、已採用 3 題、不採用 1 題、審查中 3 題
4. 下方列表顯示所有跨專案的命題記錄
5. 可用專案下拉篩選特定梯次的記錄

### 情境三：內部人員自行加入人才庫

1. 管理員（同時也是總召）需要被指派為總審人員
2. 進入教師管理系統 → 點擊【新增教師】
3. 輸入自己的內部人員信箱（帳號）
4. 系統偵測到該信箱已存在於內部人員帳號 → 自動合併
5. 提示：「此信箱已為內部人員帳號，將自動連結，使用同一組帳密登入」
6. 填寫任教背景等資料後儲存
7. 切換至「參與專案」→ 將自己加入當前梯次 → 指派為「總召(專員)」
8. 之後使用原本的內部帳密登入，即可同時使用管理功能與總審功能

## 9. 資料庫資料表規劃

### 9.1 Teachers（教師人才庫表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| UserId | UNIQUEIDENTIFIER | FK → Users.Id, NOT NULL, UNIQUE | 對應的使用者帳號（合併機制核心） |
| TeacherCode | NVARCHAR(10) | NOT NULL, UNIQUE | 教師編號（如 T1001） |
| Gender | NVARCHAR(5) | NULL | 性別 |
| Phone | NVARCHAR(20) | NULL | 聯絡電話 |
| IdNumber | NVARCHAR(20) | NULL | 身分證字號（加密儲存） |
| School | NVARCHAR(100) | NOT NULL | 任教學校 |
| Department | NVARCHAR(50) | NULL | 系所/科別 |
| Title | NVARCHAR(20) | NULL | 職稱 |
| Expertise | NVARCHAR(200) | NULL | 專長領域 |
| TeachingYears | INT | NULL | 教學年資 |
| Education | NVARCHAR(10) | NULL | 最高學歷 |
| Note | NVARCHAR(500) | NULL | 備註 |
| CreatedAt | DATETIME2 | DEFAULT GETUTCDATE() | 建立時間 |
| UpdatedAt | DATETIME2 | DEFAULT GETUTCDATE() | 最後更新時間 |

> **設計說明：** Teachers 表與 Users 表透過 `UserId` 一對一關聯。姓名、信箱、帳號狀態、首次登入等欄位統一由 Users 表管理，避免資料重複。

### 9.2 帳號合併機制對應的 Users 表關聯

```
Users 表 (1) ──── (1) Teachers 表
    │                    │
    │                    └─ 教師專屬欄位（學校、職稱、專長...）
    │
    ├─ Account (帳號/信箱)
    ├─ PasswordHash
    ├─ Name
    ├─ IsActive (帳號啟停)
    ├─ IsFirstLogin (首次登入旗標)
    └─ RoleId (系統角色)
```

**合併時：** 若 Users 表已有該 Email 的內部人員帳號，直接在 Teachers 表新增一筆記錄並指向該 UserId，無需建立新 Users 記錄。

## 10. API 端點規劃（Blazor .NET 10 遷移參考）

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/teachers` | 取得教師列表（含篩選：status/search） |
| GET | `/api/teachers/{id}` | 取得單一教師詳情 |
| POST | `/api/teachers` | 新增教師（含帳號合併檢查） |
| PUT | `/api/teachers/{id}` | 編輯教師資料 |
| PUT | `/api/teachers/{id}/account-status` | 切換帳號啟停狀態 |
| POST | `/api/teachers/{id}/reset-password` | 重設密碼為預設值 |
| GET | `/api/teachers/{id}/question-history` | 取得教師跨專案命題歷程 |
| GET | `/api/teachers/{id}/projects` | 取得教師參與的專案列表 |
| POST | `/api/teachers/{id}/projects` | 將教師加入專案並指派角色 |
| DELETE | `/api/teachers/{id}/projects/{projectId}` | 將教師從專案移除（含防呆） |
| GET | `/api/teachers/export?projectId=` | 匯出本梯次教師名單（CSV/Excel） |
| GET | `/api/teachers/stats?projectId=` | 取得統計數據（總數、啟停、本梯次） |
