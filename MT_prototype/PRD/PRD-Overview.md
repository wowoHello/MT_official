## 1. 頁面概述

命題總覽頁面提供管理者與相關權限人員檢視當前梯次所有試題的命題狀態、三審進度（7 階段燈號）與審查歷程。此頁面為**純檢視頁面**，不提供編輯功能，但可展開抽屜查看題目內容與審查意見，並可匯出結案報表。

**可檢視此頁面的角色層級：**
1. 系統管理員（ADMIN）
2. 計畫主持人
3. 總召老師
4. 教務管理者

## 4. 功能需求

### 4.1 麵包屑與標題列

- 麵包屑路徑：首頁 > 命題總覽
- 頁面標題：「當前梯次：命題總覽」
- 右上方按鈕區：**匯出報表**

### 4.2 匯出報表按鈕

- **顯示條件**：該梯次已結案（`status === 'closed'`）時才顯示此按鈕
- **匯出格式**：Excel 檔（`.xlsx`）
- **匯出欄位**：

| # | 欄位 | 說明 |
|---|------|------|
| I | 題碼 | 試題編號（含子題後綴，如 Q-2603-001-A） |
| II | 題型 | 題型名稱 |
| III | 建立日期 | 試題首次建立的時間 |
| IV | 命題教師 | 命題教師姓名 |
| V | 試題資料 | 題幹內容摘要、最後修題日期 |
| VI | 專審教師、專審結果 | 專審委員姓名、專審決策結果 |
| VII | 總審教師、總審結果 | 總審人員姓名、總審決策結果 |

- 檔名格式：`CWT_試題報表_{梯次ID}_{時間戳}.xlsx`
- 匯出需加 BOM 處理避免中文亂碼

### 4.3 狀態計數卡片（5 張）

上方顯示 5 張統計卡片，資料依當前梯次即時計算：

| # | 卡片名稱 | 計算邏輯 | 顯示樣式 |
|---|---------|---------|---------|
| 1 | 總題數 | 該梯次所有試題總數（含題組展開後的子題數） | 深灰大字 |
| 2 | 命題中 | 狀態為 `draft` 或 `completed` 的題數 | 一般大字 |
| 3 | 審核中 | 狀態為 `peer_reviewing`、`expert_reviewing`、`final_reviewing` 的題數 | 一般大字 |
| 4 | 待修編 | 狀態為 `peer_editing`、`expert_editing`、`final_editing` 的題數 | Terracotta 橘色大字 |
| 5 | 已採用 | 狀態為 `adopted` 的題數 | Sage 綠色大字 |

> ※ 目前的卡片設計須調整：由原本的 5+1 合併卡片設計，調整為上述 5 張獨立平等卡片。

### 4.4 篩選列

四維度複合篩選，篩選結果即時更新列表：

| 篩選器 | 類型 | 說明 |
|--------|------|------|
| 關鍵字搜尋 | 文字輸入 | 模糊比對題號或教師姓名 |
| 題型下拉 | Select | 所有題型 / 一般單選題 / 精選單選題 / 閱讀題組 / 長文題目 / 短文題組 / 聽力測驗 / 聽力題組 |
| 等級下拉 | Select | 所有等級 + 一般等級（初級～優級）+ 聽力等級（難度一～五） |
| 狀態下拉 | Select | 所有狀態 / 階段推進中 / 退回修題中 / 已採用入庫 / 不採用報廢 |

**等級連動邏輯：**
- 選擇聽力相關題型時 → 隱藏一般等級選項，顯示聽力難度選項
- 選擇非聽力題型時 → 顯示一般等級選項，隱藏聽力難度選項
- 選擇「所有題型」時 → 全部等級選項皆顯示
- 若當前選中的等級被隱藏 → 自動重置為「所有等級」

**篩選列右側**：顯示「共 X 題」，即時反映篩選後的結果數量。

### 4.5 試題列表表格

#### 4.5.1 表頭欄位

| 欄位 | Grid 佔比 | 說明 |
|------|----------|------|
| 題號 / 作者 | col-span-2 | 題號 + 命題教師姓名 |
| 題型 | col-span-1 | 題型 Badge |
| 等級 / 難度 | col-span-1 | 等級 Badge + 難度 Badge |
| 三審狀態軌跡（7 階段） | col-span-7 | 7 顆進度燈號 |
| 當前狀態 | col-span-1 | 狀態 Badge |

#### 4.5.2 題組呈現規則（重要設計決策）

**原則：母題 + 子題 N = 一筆獨立列表紀錄**

對於題組類型（閱讀題組 `readGroup`、短文題組 `shortGroup`、聽力題組 `listenGroup`），依子題數量展開為多筆列表項目：

| 題組結構 | 列表顯示 |
|---------|---------|
| 母題有 2 個子題 | Q-YYMM-###-A（母題+子題1）<br>Q-YYMM-###-B（母題+子題2） |
| 母題有 3 個子題 | Q-YYMM-###-A（母題+子題1）<br>Q-YYMM-###-B（母題+子題2）<br>Q-YYMM-###-C（母題+子題3） |
| 非題組（一般/精選/長文/聽力） | Q-YYMM-### （無後綴） |

**編碼規則：**
- 基礎編碼：`Q-{年月}-{流水號}`，如 `Q-2603-001`
- 子題後綴：`-A`（子題1）、`-B`（子題2）、`-C`（子題3）...依序遞增
- 題組的所有子題共用同一個母題、同一個狀態、同一條三審軌跡

**抽屜展開內容差異：**
- 點擊 `Q-2603-001-A` → 顯示**母題（閱讀文章/聽力腳本/短文）+ 子題 1** 的題幹、選項/作答區、解析
- 點擊 `Q-2603-001-B` → 顯示**母題（同上）+ 子題 2** 的題幹、選項/作答區、解析
- 歷程軌跡與審查意見為母題級別，所有子題共用

#### 4.5.3 列表項目互動

- **Hover 效果**：左側 4px Morandi 藍邊框 + 淺藍底色
- **點擊效果**：開啟右側滑入抽屜面板
- **特殊標記**：退回次數 ≥ 3 時，題號旁顯示紅色脈衝「強制」Badge

### 4.6 三審狀態軌跡（7 階段燈號）

7 顆燈號橫向排列，代表試題生命週期中的 7 個階段：

| # | 階段代碼 | 顯示標籤 |
|---|---------|---------|
| 1 | 命題 | 命題 |
| 2 | 互審 | 互審 |
| 3 | 互修 | 互修 |
| 4 | 專審 | 專審 |
| 5 | 專修 | 專修 |
| 6 | 總審 | 總審 |
| 7 | 總修 | 總修 |

**燈號顏色與圖示規則：**

| 狀態 | 圓點顏色 | 圖示 | 動畫 |
|------|---------|------|------|
| 已通過 | Sage 綠 `#8EAB94` | ✓ 勾勾 | 無 |
| 執行中（命題中/審題中） | Morandi 藍 `#6B8EAD` | ⟳ Spinner | 藍色脈衝光暈 |
| 修題中（退回修改） | Terracotta 橘 `#D98A6C` | ✎ 筆 | 紅色脈衝光暈 |
| 不採用停止 | Terracotta 橘 | ✕ 叉 | 無 |
| 未到達 | 灰色 | 階段序號 | 無 |
| 已採用（全通過） | 全部 Sage 綠 | 全部 ✓ | 無 |

**與階段時間連動：**
> ※ 燈號跟著實際設定的階段時間區間跑。例如：當梯次進入「互審階段」區間時，所有已完成命題的試題，命題燈號一定是 ✓（通過）或 ✕（截止時仍為草稿則視為未完成）。

### 4.7 抽屜面板（Slide-Over Detail）

從右側滑入的詳情面板（`max-w-3xl`），分為上下兩大區塊：

#### 4.7.1 面板標頭

| 欄位 | 說明 |
|------|------|
| 題號 | 如 `Q-2603-001-A` |
| 狀態 Badge | 如「專審中」 |
| 警告 Badge | 退回次數 ≥ 3 時顯示「總召強制收回」紅色脈衝 |
| 關閉按鈕 | 點擊或點擊背景遮罩關閉 |

#### 4.7.2 上半部：題目內容區塊

**顯示順序：題目內容必須顯示於審題結果之前**

| 資訊 | 位置 |
|------|------|
| 命題教師 | 左上 |
| 題型 / 等級 / 難易度 | 右上 |
| 題幹內容區 | 主體 |

**依題型不同的內容呈現：**

| 題型 | 呈現內容 |
|------|---------|
| 一般單選題 / 精選單選題 | 題幹 + 4 選項（正確答案綠底標記）+ 解析 |
| 長文題目 | 作文題幹（琥珀色框）+ 解析 |
| 閱讀題組（子題 N） | 閱讀文章（橘色標籤 + 灰底框）+ **對應子題 N** 的題幹與選項 + 題組解析 |
| 短文題組（子題 N） | 短文內容（綠色標籤 + 灰底框）+ **對應子題 N** 的論述題題幹 + 子題解析 |
| 聽力測驗 | 音檔播放區（藍底框）+ 題幹 + 4 選項 + 解析 |
| 聽力題組（子題 N） | 音檔播放區 + 聽力腳本（粉色標籤 + 灰底框）+ **對應子題 N** 的題幹與選項 + 題組解析 |

#### 4.7.3 下半部：歷程軌跡與審查意見

- 以 Timeline 格式呈現，左側時間軸線
- **排序：依日期排序，新 → 舊**（最新紀錄在最上方）
- 最新一筆以 Morandi 藍圓點 + 陰影邊框突出

每筆紀錄包含：

| 欄位 | 說明 |
|------|------|
| 操作人員 | 姓名 + 角色圖示（教師綠/委員藍/總召橘） |
| 動作標籤 | 如「命題完成」「互審意見」「專審意見（採用）」「總召決策（改後再審）」 |
| 時間 | 右對齊，等寬字型 |
| 審查意見 | 引號框包裹的意見內容（有則顯示） |

**動作標籤顏色：**

| 動作類型 | 背景/文字色 | 圖示 |
|---------|------------|------|
| 含「採用」（非「不採用」） | 綠底綠字 | ✓ 勾勾 |
| 含「不採用」 | 灰底灰字 | ⊘ 禁止 |
| 含「改後再審」或「退回」 | 橘底橘字 | ↺ 退回 |
| 其他 | 白底灰字 | 無 |

## 5. 試題狀態機

### 5.1 完整狀態定義

| 狀態代碼 | 顯示標籤 | Badge 色 |
|---------|---------|---------|
| `draft` | 草稿 | 灰 |
| `completed` | 命題完成 | 藍 |
| `pending` | 待審 | 黃 |
| `peer_reviewing` | 互審中 | Morandi 藍 |
| `peer_reviewed` | 互審完成 | 綠 |
| `peer_editing` | 互審修題 | Terracotta 橘 |
| `expert_reviewing` | 專審中 | Morandi 藍 |
| `expert_reviewed` | 專審完成 | 綠 |
| `expert_editing` | 專審修題 | Terracotta 橘 |
| `final_reviewing` | 總審中 | Morandi 藍 |
| `final_reviewed` | 總審完成 | 紅 |
| `final_editing` | 總審修題 | Terracotta 橘 |
| `adopted` | 採用 | Sage 綠 |
| `rejected` | 不採用 | 灰 |

### 5.2 狀態流轉圖

```
draft → completed → pending
    ↓
peer_reviewing → peer_reviewed
    ↓ (退回)
peer_editing → (回到 peer_reviewing 或進入下階段)
    ↓
expert_reviewing → expert_reviewed
    ↓ (退回)
expert_editing → (回到 final_reviewing，注意：專審退回修完直接進總審)
    ↓
final_reviewing → final_reviewed
    ↓ (退回，最多2次)          ↓ (採用)
final_editing → (回到 final_reviewing)   adopted（入庫）
    ↓ (第3次不通過)
    總召自行修改 → adopted 或 rejected
```

## 6. 題組編號規則（設計決策）

### 6.1 編號格式

```
基礎格式：Q-{年月}-{流水號}
題組格式：Q-{年月}-{流水號}-{子題代碼}

範例：
  Q-2603-001      ← 一般單選題（無子題）
  Q-2603-002-A    ← 閱讀題組，母題 + 子題 1
  Q-2603-002-B    ← 閱讀題組，母題 + 子題 2
  Q-2603-003      ← 聽力測驗（無子題）
  Q-2603-004-A    ← 聽力題組，母題 + 子題 1
  Q-2603-004-B    ← 聽力題組，母題 + 子題 2
  Q-2603-004-C    ← 聽力題組，母題 + 子題 3
```

### 6.2 題組在列表中的呈現邏輯

| 面向 | 行為 |
|------|------|
| 列表顯示 | 題組的每個子題各佔一列，編號帶後綴 |
| 題型欄位 | 所有子題均顯示母題的題型（如「閱讀題組」） |
| 等級/難度 | 所有子題共用母題的等級與難度 |
| 7 階段燈號 | 所有子題共用母題的進度狀態 |
| 當前狀態 | 所有子題共用母題的當前狀態 |
| 點擊展開 | 展示母題內容 + 對應子題 N 的題幹與解析 |
| 歷程軌跡 | 所有子題共用母題的審查歷程 |
| 計數（總題數） | 每個子題計為 1 題（2 子題 = 2 題） |

### 6.3 排序規則

列表按以下優先順序排序：
1. 流水號遞增
2. 子題代碼遞增（A → B → C ...）

## 7. 使用情境

### 情境一：管理員搜尋特定教師的命題

> 假設目前進行中的產學合作計畫為：115年度春季全民中檢

1. 管理員進入命題總覽頁面
2. 在搜尋框輸入教師姓名（模糊搜尋）
3. 列表即時篩選顯示該教師的所有命題
4. 找到 `Q-2603-001-A`（閱讀題組），點擊該列
5. 右側抽屜滑入，顯示：
   - 閱讀文章（母題）
   - 子題 1 的題幹與選項
   - 解析
   - 歷程軌跡與審查意見（新→舊排序）
6. 關閉抽屜，找到 `Q-2603-001-B`，點擊該列
7. 右側抽屜滑入，顯示：
   - 閱讀文章（同一母題）
   - 子題 2 的題幹與選項
   - 解析
   - 歷程軌跡（與 -A 共用同一份）

## 8. 資料庫資料表規劃

### 8.1 Questions（試題母題表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| ProjectId | UNIQUEIDENTIFIER | FK → Projects.Id, NOT NULL | 所屬梯次 |
| QuestionCode | NVARCHAR(20) | NOT NULL, UNIQUE | 題碼（如 `Q-2603-001`） |
| QuestionTypeCode | NVARCHAR(20) | NOT NULL | 題型代碼 |
| Level | NVARCHAR(10) | NOT NULL | 等級（初級/中級/...或難度一～五） |
| Difficulty | NVARCHAR(10) | NOT NULL | 難易度（easy/medium/hard） |
| AuthorId | UNIQUEIDENTIFIER | FK → Users.Id, NOT NULL | 命題教師 |
| Stem | NVARCHAR(MAX) | NULL | 題幹（非題組類型）或作文題目 |
| Passage | NVARCHAR(MAX) | NULL | 閱讀文章/短文/聽力腳本（題組母題用） |
| AudioUrl | NVARCHAR(500) | NULL | 聽力音檔路徑（聽力題型用） |
| Options | NVARCHAR(MAX) | NULL | 選項 JSON（非題組類型：`[{label, text}]`） |
| Answer | NVARCHAR(10) | NULL | 正確答案代碼（如 A/B/C/D） |
| Analysis | NVARCHAR(MAX) | NULL | 解析（非題組類型）或題組總體解析 |
| CurrentStage | INT | NOT NULL, DEFAULT 1 | 當前所在階段（1~7） |
| Status | NVARCHAR(30) | NOT NULL, DEFAULT 'draft' | 當前狀態代碼 |
| ReturnCount | INT | NOT NULL, DEFAULT 0 | 累計退回次數 |
| CreatedAt | DATETIME2 | DEFAULT GETUTCDATE() | 建立時間 |
| UpdatedAt | DATETIME2 | DEFAULT GETUTCDATE() | 最後更新時間 |

### 8.2 SubQuestions（子題表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| QuestionId | UNIQUEIDENTIFIER | FK → Questions.Id, NOT NULL | 所屬母題 |
| SubIndex | INT | NOT NULL | 子題序號（1, 2, 3...） |
| SubCode | CHAR(1) | NOT NULL | 子題代碼後綴（A, B, C...） |
| Stem | NVARCHAR(MAX) | NOT NULL | 子題題幹 |
| Options | NVARCHAR(MAX) | NULL | 選項 JSON（選擇題：`[{label, text}]`，論述題為 NULL） |
| Answer | NVARCHAR(10) | NULL | 正確答案代碼（論述題為 NULL） |
| Analysis | NVARCHAR(MAX) | NULL | 子題解析 |
| SortOrder | INT | NOT NULL, DEFAULT 0 | 排序用 |

**UNIQUE 約束：** `(QuestionId, SubIndex)` 聯合唯一

### 8.3 QuestionHistoryLogs（試題歷程軌跡表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| QuestionId | UNIQUEIDENTIFIER | FK → Questions.Id, NOT NULL | 所屬母題（歷程為母題級別） |
| UserId | UNIQUEIDENTIFIER | FK → Users.Id, NOT NULL | 操作人員 |
| Action | NVARCHAR(50) | NOT NULL | 動作（如「命題完成」「互審意見」「專審意見（採用）」） |
| Comment | NVARCHAR(MAX) | NULL | 審查意見內容 |
| CreatedAt | DATETIME2 | DEFAULT GETUTCDATE() | 操作時間 |

**INDEX：** `QuestionId` + `CreatedAt DESC`（支援依日期新→舊排序查詢）

### 8.4 QuestionTypes（題型設定表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Code | NVARCHAR(20) | PK | 題型代碼 |
| Name | NVARCHAR(20) | NOT NULL | 題型中文名稱 |
| HasSubQuestions | BIT | NOT NULL, DEFAULT 0 | 是否為題組類型 |
| SubQuestionMode | NVARCHAR(20) | NULL | 子題模式（`choice`=選擇題/`freeResponse`=論述題/NULL） |
| HasAudio | BIT | NOT NULL, DEFAULT 0 | 是否有音檔 |
| HasPassage | BIT | NOT NULL, DEFAULT 0 | 是否有母題閱讀段落 |
| PassageLabel | NVARCHAR(20) | NULL | 段落標籤（如「閱讀文章」「短文內容」「聽力腳本」） |

**預設資料：**

| Code | Name | HasSubQuestions | SubQuestionMode | HasAudio | HasPassage |
|------|------|----------------|----------------|----------|------------|
| `single` | 一般單選題 | 0 | NULL | 0 | 0 |
| `select` | 精選單選題 | 0 | NULL | 0 | 0 |
| `longText` | 長文題目 | 0 | NULL | 0 | 0 |
| `readGroup` | 閱讀題組 | 1 | choice | 0 | 1 |
| `shortGroup` | 短文題組 | 1 | freeResponse | 0 | 1 |
| `listen` | 聽力測驗 | 0 | NULL | 1 | 0 |
| `listenGroup` | 聽力題組 | 1 | choice | 1 | 1 |

## 9. API 端點規劃（Blazor .NET 10 遷移參考）

| 方法 | 路徑 | 說明 |

## 10. 已知限制與改進方向

| 項目 | 現況（Demo） | 遷移後目標 |
|------|-------------|-----------|
| 試題資料 | 前端寫死 `mockQuestionsDb` | 後端 API + 資料庫 |
| 題組呈現 | 目前為母題一列（未展開子題） | 按子題展開為多列（Q-###-A、Q-###-B） |
| 匯出格式 | CSV 格式（僅基礎欄位） | Excel（xlsx）含完整 7 欄匯出 |
| 匯出按鈕顯示 | 始終顯示 | 僅結案後顯示 |
| 統計卡片 | 5+1 合併設計 | 調整為 5 張獨立平等卡片 |
| 燈號與階段時間連動 | 固定模擬階段值 | 依實際階段時間區間動態判定 |
| 抽屜題組顯示 | 顯示母題 + 所有子題 | 僅顯示母題 + 對應子題 N |
| 教師名稱 | 前端寫死 teacherMap | 後端 Join 查詢 |
| 分頁 | 無（全部載入） | 後端分頁（PageSize + PageIndex） |
