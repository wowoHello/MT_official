## 1. 頁面概述

命題專案管理頁面提供系統管理員對產學合作命題專案（梯次）的完整生命週期管理，包含新增、編輯、檢視、結案入庫等核心操作。頁面採左右分割佈局，左側為梯次列表（含篩選），右側為選中梯次的詳細資訊。

**可檢視此頁面的角色：** 系統管理員（ADMIN）

## 4. 功能需求

### 4.1 麵包屑導覽

- 路徑：首頁 > 命題專案管理
- 「首頁」可點擊返回 `firstpage.html`

### 4.2 左側：梯次列表面板

#### 4.2.1 搜尋框

- 即時搜尋過濾：比對專案名稱或年度
- 搜尋圖示 + placeholder「搜尋專案名稱 / 年度...」

#### 4.2.2 狀態篩選按鈕

三個篩選按鈕，互斥切換：

| 篩選 | 過濾邏輯 | 預設樣式 |
|------|---------|---------|
| 全部 | 顯示所有梯次 | 預設選中（Morandi 藍底白字） |
| 進行中 | 顯示 `status === 'active'` 或 `'preparing'` | 白底灰框 |
| 已結案 | 顯示 `status === 'closed'` | 白底灰框 |

#### 4.2.3 梯次列表項目

每個列表項目顯示：
- 專案名稱（粗體、單行截斷）
- 狀態 Badge：進行中（藍）/ 準備中（琥珀黃）/ 已結案（灰）
- 年度 / 合作學校
- 右側箭頭指示

選中項目樣式：左側 4px Morandi 藍邊框 + 淺藍背景 + 文字變色

無結果時：「查無命題專案」

### 4.3 右側：梯次詳細資訊面板

#### 4.3.1 基本資料卡片

| 顯示欄位 | 說明 |
|---------|------|
| 狀態 Badge | 進行中 / 準備中 / 已結案 |
| 年度 | 如「115年度」 |
| 專案名稱 | 如「115年度 春季全民中檢」 |
| 合作學校 | 如「無（自辦）」或學校名稱 |
| 建立者 | 如「系統管理員」 |

**操作按鈕（已結案時隱藏）：**

| 按鈕 | 說明 |
|------|------|
| 編輯專案 | 開啟右側滑入面板進入編輯模式 |
| 結案入庫 | 執行結案流程（兩次確認） |

#### 4.3.2 時程規劃設定（Timeline 呈現）

以行程路線（Timeline）的方式顯示 8 階段時程，每個節點包含：
- 圓點指示球
- 階段序號 + 名稱
- 起迄日期

**圓點顏色規則：**

| 狀態 | 條件 | 圓點顏色 | 連線樣式 |
|------|------|---------|---------|
| 已完成 | `EndDate < 今天`（進行中專案） | Sage 綠 `#8EAB94` | 綠色實線 |
| 進行中 | `StartDate ≤ 今天 ≤ EndDate` | Morandi 藍 + 藍色外環光暈 | 灰色虛線 |
| 未開始 | `StartDate > 今天` | 灰色 | 灰色虛線 |
| 已完成（結案） | `EndDate < 結案日`（已結案專案） | Morandi 藍 | 藍色實線 |
| 被強制結案中斷 | 結案日落在此階段區間內 | Terracotta 紅橘 + 紅色外環 | 灰色虛線 |
| 未完成被結案跳過 | `StartDate > 結案日` | 灰色 | 灰色虛線 |

#### 4.3.3 題型目標數量

以格線（`grid-cols-2 sm:grid-cols-3`）排列顯示 7 種題型的目標數量：

| 題型代碼 | 題型名稱 |
|---------|---------|
| `single` | 一般單選題 |
| `select` | 精選單選題 |
| `readGroup` | 閱讀題組 |
| `longText` | 長文題目 |
| `shortGroup` | 短文題組 |
| `listen` | 聽力測驗 |
| `listenGroup` | 聽力題組 |

每格顯示題型名稱（小字）+ 目標數量（大數字粗體）。

#### 4.3.4 指派人員列表

- 標題列顯示「共 X 人」Badge
- 可捲動列表（`max-h: 320px`）
- 每位人員顯示：

#### 4.5.2 區塊一：基本設定

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| 所屬年度 | 文字 | ✓ | 如「115」 |
| 專案名稱 | 文字 | ✓ | 如「春季全民中檢」 |
| 合作學校 | 文字 | ✗ | 留白表示自辦梯次 |

- 新增時狀態預設為「準備中」（`preparing`）
- 不允許直接設為「已結案」

#### 4.5.3 區塊二：時程規劃設定（8 階段連動日期引擎）

8 個階段各有起始日與結束日輸入框，具備**自動推算連動引擎**：

**操作提示：** 只需設定第一階段「產學計畫區間」的開始日，系統即自動推算半年內的 7 階段日程。手動變更任一階段的結束日，會自動將下一階段的開始日連動推進為隔天。

**8 階段定義與預設天數：**

| # | 階段名稱 | 預設天數 | 倒數5天提醒 |
|---|---------|---------|------------|
| 1 | 產學計畫區間 | 約 100 天 | ✗ |
| 2 | 命題階段 | 30 天 | ✓ |
| 3 | 交互審題 | 7 天 | ✗ |
| 4 | 互審修題 | 7 天 | ✓ |
| 5 | 專家審題 | 14 天 | ✗ |
| 6 | 專審修題 | 14 天 | ✓ |
| 7 | 總召審題 | 14 天 | ✗ |
| 8 | 總召修題 | 14 天 | ✓ |

**連動引擎邏輯：**
1. 設定階段 1 的開始日 → 自動推算階段 1 的結束日（+100 天）
2. 階段 2 開始日 = 階段 1 開始日 + 14 天
3. 每階段結束日變更 → 下一階段開始日 = 該結束日 + 1 天
4. 所有日期皆可手動覆寫

#### 4.5.4 區塊三：目標題數與人員指派

**目標題數：**
- 7 種題型各一個數字輸入框（`type="number"`）
- 格線排列（`grid-cols-2 sm:grid-cols-4 lg:grid-cols-7`）

**人員指派：**
- 人員列表來源：教師管理系統人才庫
- 身份別來源：角色與權限管理
- 上方搜尋框：可搜尋姓名或編號
- 每位人員包含：
  - Checkbox 勾選（啟用/停用）
  - 姓名與編號
  - 角色下拉選單（主要身分，最多可新增至 3 個角色）
  - 勾選時啟用角色選擇，未勾選時停用（灰色）

**可指派角色：**
- 命題教師
- 互審教師
- 專審委員
- 總召(專員)
- 內部人員

#### 4.5.5 區塊四：人員命題數量配置

- 僅顯示被指派為「命題教師」角色的人員
- 每位教師一列，顯示 7 種題型的配額輸入框
- **平均分配按鈕**：「依命題教師人數平均分配」按鈕
  - 點擊後將各題型目標數量平均分配給所有命題教師
  - 餘數分配給第一位教師
- **配額檢核**：
  - 底部顯示 7 格迷你檢核方塊（已分配 / 需分配）
  - 吻合時顯示綠色勾號，不吻合時顯示紅色提示
  - 整體驗證訊息文字

#### 4.5.6 儲存邏輯

- 表單驗證：必填欄位檢查（`checkValidity()`）
- 配額驗證：檢查配額加總是否與目標題數吻合
  - 吻合 → 直接儲存
  - 不吻合 → SweetAlert2 警示，提供「強制儲存」或「返回修改」選擇
- 儲存成功：Toast 提示 → 關閉面板 → 重新渲染列表

### 4.6 結案入庫功能

**結案業務邏輯：**
- 結案後專案狀態改為 `closed`，凍結所有操作（編輯按鈕與結案按鈕隱藏）
- 僅「採用」狀態的題目會正式入庫，其餘一律標記為「不採用」並鎖定
- 結案後不可變更

**兩次確認防呆機制：**

| 情境 | 觸發條件 | 確認內容 |
|------|---------|---------|
| 提早結案 | 產學計畫結束日（階段1 EndDate）尚未到達 | ⚠️ 「期程尚未結束，確定要提早結案嗎？」+ 強制結案按鈕 |
| 正常結案 | 產學計畫結束日已過 | ❓ 「確定結案入庫？僅採用狀態的題目會入庫。」 |

結案後記錄實際結案日期（`closedDate`），用於時程 Timeline 凍結顯示。

## 6. 使用情境

### 情境：系統管理員管理命題專案

> 假設目前進行中的產學合作計畫為：115年度春季全民中檢

1. **指派人員與分配題目**：管理員想新增指派人員並將題目平均分配給教師命題，以及將題目分配給審題人員審題 → 點擊右側**編輯專案**按鈕 → 滑入面板開啟 → 在區塊三勾選人員並指派角色 → 在區塊四使用「平均分配」或手動調整命題配額 → 儲存

2. **查看已結案梯次**：管理員想查看歷史梯次 → 在左側點擊**已結案**篩選按鈕 → 列表過濾顯示已結案梯次 → 點擊任一梯次檢視詳情（Timeline 顯示完成或中斷狀態）

3. **提前結案**：管理員確認所有事項無誤後想提前結案 → 點擊右側**結案入庫**按鈕 → 系統偵測到產學結束日尚未到達，彈出警示確認「期程尚未結束，確定要強制結案嗎？」→ 確認後專案狀態改為已結案 → 除「採用」狀態外的題目全部改為「不採用」→ 編輯與結案按鈕隱藏

4. **新增新梯次**：管理員想建立「115年度冬季全民中檢」→ 點擊右上**新增梯次**按鈕 → 滑入面板開啟（新增模式）→ 填寫基本設定、時程、目標題數、人員指派 → 儲存建立

## 7. 資料庫資料表規劃

> 本頁面涉及的主要資料表已在 PRD-FirstPage 定義（Projects、ProjectPhases、ProjectMembers）。以下補充本頁面特有的配額與結案相關欄位。

### 7.1 Projects 表補充欄位

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| CreatedBy | UNIQUEIDENTIFIER | FK → Users.Id, NOT NULL | 建立者 |

### 7.2 ProjectTargets（專案題型目標數量表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| ProjectId | UNIQUEIDENTIFIER | FK → Projects.Id, NOT NULL | 所屬專案 |
| QuestionTypeCode | NVARCHAR(20) | NOT NULL | 題型代碼（`single`、`select`、`readGroup`、`longText`、`shortGroup`、`listen`、`listenGroup`） |
| TargetCount | INT | NOT NULL, DEFAULT 0 | 目標題數 |

**UNIQUE 約束：** `(ProjectId, QuestionTypeCode)` 聯合唯一

### 7.3 MemberQuotas（人員命題配額表）

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| ProjectMemberId | UNIQUEIDENTIFIER | FK → ProjectMembers.Id, NOT NULL | 所屬專案成員 |
| QuestionTypeCode | NVARCHAR(20) | NOT NULL | 題型代碼 |
| Quota | INT | NOT NULL, DEFAULT 0 | 分配的命題數量 |

**UNIQUE 約束：** `(ProjectMemberId, QuestionTypeCode)` 聯合唯一

### 7.4 ProjectMembers 表補充欄位

> 一位人員在同一梯次中可能身兼多個角色（如同時為命題教師與互審教師），因此 ProjectMembers 中的 AssignedRoleCode 需支援多角色。

**方案一（推薦）：** 新增 ProjectMemberRoles 關聯表

| 欄位名稱 | 資料型態 | 約束 | 說明 |
|----------|---------|------|------|
| Id | UNIQUEIDENTIFIER | PK, DEFAULT NEWID() | 唯一識別碼 |
| ProjectMemberId | UNIQUEIDENTIFIER | FK → ProjectMembers.Id, NOT NULL | 專案成員 |
| RoleCode | NVARCHAR(20) | NOT NULL | 角色代碼（`TEACHER`、`CROSS_REVIEWER`、`EXPERT_REVIEWER`、`CHIEF`、`INTERNAL`） |

## 8. API 端點規劃（Blazor .NET 10 遷移參考）

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | `/api/projects` | 取得所有專案列表（含篩選參數 `?status=active`） |
| GET | `/api/projects/{id}` | 取得單一專案詳情（含階段、目標、人員） |
| POST | `/api/projects` | 新增專案 |
| PUT | `/api/projects/{id}` | 編輯專案（基本資料、階段、目標、人員） |
| POST | `/api/projects/{id}/close` | 結案入庫（含提早結案檢查） |
| GET | `/api/projects/{id}/members` | 取得專案指派人員列表 |
| PUT | `/api/projects/{id}/members` | 更新專案指派人員與配額 |
| POST | `/api/projects/{id}/members/auto-distribute` | 自動平均分配命題配額 |
| GET | `/api/teachers` | 取得教師人才庫（供人員指派選擇） |
